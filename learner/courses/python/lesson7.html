Lesson 7: Lists & Tuples

Work with ordered collections of items using lists and tuples

What are Lists?

Lists are ordered, mutable collections that can hold items of any type. They're one of Python's most versatile data structures, perfect for storing sequences of related items. Lists can grow and shrink dynamically.

Creating Lists

Create lists using square brackets or the list() constructor. Lists can contain any type of data, including other lists.

List Creation:

# Empty list
empty = []
also_empty = list()

# List with items
numbers = [1, 2, 3, 4, 5]
fruits = ["apple", "banana", "cherry"]

# Mixed types (allowed but usually avoid)
mixed = [1, "hello", 3.14, True, None]

# List from other iterables
from_string = list("hello")  # ['h', 'e', 'l', 'l', 'o']
from_range = list(range(5))  # [0, 1, 2, 3, 4]

# Nested lists (2D list)
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]
print(matrix[1][2])  # 6 (row 1, column 2)

# List repetition
zeros = [0] * 5
print(zeros)  # [0, 0, 0, 0, 0]

# List concatenation
combined = [1, 2] + [3, 4]
print(combined)  # [1, 2, 3, 4]

Lines 2-3: Two ways to create empty lists.

Lines 6-7: Lists with initial values using square brackets.

Lines 10-11: Lists can contain mixed types, but this is usually not recommended.

Lines 14-15: Convert other iterables to lists using list().

Lines 18-24: Nested lists create multi-dimensional structures.

Lines 27-31: Lists support * for repetition and + for concatenation.

Accessing List Elements

Access list items using indices (0-based) or slices. Negative indices count from the end of the list.

Indexing and Slicing:

fruits = ["apple", "banana", "cherry", "date", "elderberry"]

# Positive indexing (starts at 0)
print(fruits[0])   # apple (first item)
print(fruits[2])   # cherry (third item)

# Negative indexing (starts at -1)
print(fruits[-1])  # elderberry (last item)
print(fruits[-2])  # date (second to last)

# Slicing: list[start:end:step]
print(fruits[1:3])    # ['banana', 'cherry']
print(fruits[:3])     # ['apple', 'banana', 'cherry']
print(fruits[2:])     # ['cherry', 'date', 'elderberry']
print(fruits[::2])    # ['apple', 'cherry', 'elderberry']
print(fruits[::-1])   # Reversed list!

# Slicing creates new lists (copies)
copy_fruits = fruits[:]
print(copy_fruits is fruits)  # False (different objects)

# Index out of range error
# print(fruits[10])  # IndexError!

# Safe access with default
def get_item(lst, index, default=None):
    if 0 <= index < len(lst):
        return lst[index]
    return default

print(get_item(fruits, 10, "Not found"))  # Not found

Lines 4-5: Positive indices start at 0 from the beginning.

Lines 8-9: Negative indices count backward from the end (-1 is last).

Lines 12-16: Slicing extracts a portion: [start:end:step]. End is exclusive.

Line 17: [::-1] creates a reversed copy of the list.

Lines 20-21: Slicing always creates a new list object.

Lines 27-31: Handle index errors gracefully with bounds checking.

Modifying Lists

Lists are mutable, meaning you can change their contents without creating a new list. You can modify, add, and remove elements.

List Modification:

numbers = [1, 2, 3, 4, 5]

# Modify by index
numbers[0] = 10
print(numbers)  # [10, 2, 3, 4, 5]

# Modify with slice
numbers[1:3] = [20, 30]
print(numbers)  # [10, 20, 30, 4, 5]

# Append to end
numbers.append(6)
print(numbers)  # [10, 20, 30, 4, 5, 6]

# Insert at position
numbers.insert(0, 0)
print(numbers)  # [0, 10, 20, 30, 4, 5, 6]

# Extend with another iterable
numbers.extend([7, 8, 9])
print(numbers)  # [0, 10, 20, 30, 4, 5, 6, 7, 8, 9]

# Remove by value
fruits = ["apple", "banana", "cherry", "banana"]
fruits.remove("banana")  # Removes first occurrence
print(fruits)  # ['apple', 'cherry', 'banana']

# Remove by index with pop
last = fruits.pop()  # Removes and returns last
print(last)     # banana
print(fruits)   # ['apple', 'cherry']

item = fruits.pop(0)  # Remove at index
print(item)     # apple

# Delete with del keyword
nums = [1, 2, 3, 4, 5]
del nums[2]     # Delete at index
del nums[0:2]   # Delete slice

# Clear all elements
nums.clear()
print(nums)  # []

Lines 4-5: Assign to an index to change that element.

Lines 8-9: Assign to a slice to replace multiple elements at once.

Lines 12-13: append() adds to the end.

Lines 16-17: insert(index, value) adds at a specific position.

Lines 20-21: extend() adds all items from another iterable.

Lines 24-26: remove() deletes the first matching value.

Lines 29-35: pop() removes by index and returns the value.

Lines 38-40: del can delete items or slices.

List Methods

Python provides many built-in methods for working with lists efficiently.

Useful List Methods:

# Sorting
numbers = [3, 1, 4, 1, 5, 9, 2, 6]

# sort() modifies in place
numbers.sort()
print(numbers)  # [1, 1, 2, 3, 4, 5, 6, 9]

# sort reverse
numbers.sort(reverse=True)
print(numbers)  # [9, 6, 5, 4, 3, 2, 1, 1]

# sorted() returns new list
original = [3, 1, 4, 1, 5]
sorted_copy = sorted(original)
print(original)     # [3, 1, 4, 1, 5] (unchanged)
print(sorted_copy)  # [1, 1, 3, 4, 5]

# Sorting with key function
words = ["apple", "pie", "a", "longer"]
words.sort(key=len)  # Sort by length
print(words)  # ['a', 'pie', 'apple', 'longer']

# Reversing
numbers = [1, 2, 3, 4, 5]
numbers.reverse()  # In place
print(numbers)  # [5, 4, 3, 2, 1]

# Counting occurrences
letters = ['a', 'b', 'a', 'c', 'a', 'b']
print(letters.count('a'))  # 3

# Finding index
print(letters.index('b'))    # 1 (first occurrence)
print(letters.index('b', 2)) # 5 (start searching from index 2)

# Copy
original = [1, 2, 3]
copy1 = original.copy()
copy2 = original[:]
copy3 = list(original)
# All create shallow copies

Lines 4-7: sort() sorts the list in place (modifies original).

Lines 13-16: sorted() returns a new sorted list, leaving original unchanged.

Lines 19-21: The key parameter accepts a function for custom sorting.

Lines 24-26: reverse() reverses in place, or use slicing [::-1].

Lines 29-30: count() returns how many times an item appears.

Lines 33-34: index() finds the position of an item.

Introduction to Tuples

Tuples are ordered, immutable sequences. Once created, they cannot be modified. Use tuples for data that shouldn't change, like coordinates or configuration values.

Creating and Using Tuples:

# Creating tuples
point = (3, 4)
single = (1,)       # Note the comma for single item!
empty = ()
coordinates = tuple([1, 2])  # From iterable

# Tuple packing and unpacking
person = ("Alice", 25, "Engineer")
name, age, job = person  # Unpacking

# Accessing elements (like lists)
print(point[0])    # 3
print(point[-1])   # 4

# Slicing (returns new tuple)
print(point[:1])   # (3,)

# Tuples are immutable
# point[0] = 5  # TypeError!

# But can contain mutable objects
data = ([1, 2], [3, 4])
data[0].append(3)  # This works!
print(data)  # ([1, 2, 3], [3, 4])

# Tuple methods
letters = ('a', 'b', 'a', 'c')
print(letters.count('a'))  # 2
print(letters.index('b'))  # 1

# Named tuples for readable code
from collections import namedtuple
Point = namedtuple('Point', ['x', 'y'])
p = Point(3, 4)
print(p.x, p.y)  # 3 4
print(p[0], p[1])  # 3 4 (still works like tuple)

Lines 2-5: Create tuples with parentheses or tuple(). Single item needs trailing comma.

Lines 8-9: Tuple packing creates a tuple; unpacking extracts values to variables.

Lines 12-15: Tuples support indexing and slicing like lists.

Lines 18-19: Tuples themselves can't be modified, but mutable contents can.

Lines 26-28: Tuples only have count() and index() methods.

Lines 31-35: Named tuples provide named access while being immutable.

💡 When to Use Tuples vs Lists

Use tuples for fixed data that shouldn't change (coordinates, RGB colors, database records). Use lists for collections that need to grow or change.

Common List Operations

Python provides built-in functions and techniques for common list operations like searching, filtering, and combining.

Practical Operations:

# Checking membership
fruits = ["apple", "banana", "cherry"]
print("apple" in fruits)    # True
print("grape" in fruits)    # False

# Finding min, max, sum
numbers = [3, 1, 4, 1, 5, 9, 2, 6]
print(min(numbers))   # 1
print(max(numbers))   # 9
print(sum(numbers))   # 31
print(len(numbers))   # 8

# List concatenation alternatives
a = [1, 2, 3]
b = [4, 5, 6]

# Using + (creates new list)
c = a + b  # [1, 2, 3, 4, 5, 6]

# Using extend (modifies a)
a_copy = a.copy()
a_copy.extend(b)

# Using unpacking (Python 3.5+)
c = [*a, *b]  # [1, 2, 3, 4, 5, 6]

# Flatten nested lists
nested = [[1, 2], [3, 4], [5, 6]]
flat = [item for sublist in nested for item in sublist]
print(flat)  # [1, 2, 3, 4, 5, 6]

# Zip lists together
names = ["Alice", "Bob", "Charlie"]
ages = [25, 30, 35]
pairs = list(zip(names, ages))
print(pairs)  # [('Alice', 25), ('Bob', 30), ('Charlie', 35)]

# Unzip
names_again, ages_again = zip(*pairs)

# Enumerate for index and value
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

Lines 3-4: Use in to check if an item exists in a list.

Lines 7-10: Built-in functions: min, max, sum, len.

Lines 19-20: Unpacking with * in list literal for combining.

Lines 23-25: Flatten 2D lists with nested comprehension.

Lines 28-30: zip() combines multiple lists element by element.

Lines 36-37: enumerate() gives both index and value in loops.

Copying Lists

Understanding the difference between shallow and deep copies is crucial when working with nested lists or objects.

Copy vs Reference:

# Reference (not a copy!)
original = [1, 2, 3]
reference = original
reference.append(4)
print(original)  # [1, 2, 3, 4] - original changed!

# Shallow copy
original = [1, 2, [3, 4]]
shallow = original.copy()  # or original[:] or list(original)

# Modify top level - OK
shallow.append(5)
print(original)  # [1, 2, [3, 4]] - unchanged

# Modify nested - Problem!
shallow[2].append(5)
print(original)  # [1, 2, [3, 4, 5]] - changed!

# Deep copy (for nested structures)
import copy
original = [1, 2, [3, 4]]
deep = copy.deepcopy(original)
deep[2].append(5)
print(original)  # [1, 2, [3, 4]] - unchanged

# When you need a truly independent copy
matrix = [[1, 2], [3, 4]]
matrix_copy = [row[:] for row in matrix]  # Manual deep copy for 2D

Lines 3-6: Assigning a list to a new variable just creates another reference.

Lines 9-10: copy(), [:], or list() create shallow copies.

Lines 13-14: Shallow copies share nested objects with the original.

Lines 17-19: Modifying nested items affects both the copy and original.

Lines 22-25: copy.deepcopy() creates completely independent copies.

Lines 28-29: For 2D lists, manually copy each row for efficiency.
