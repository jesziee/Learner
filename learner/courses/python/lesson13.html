Lesson 13: Modules & Packages

Organize code with modules and use external packages with pip

What are Modules?

Modules are Python files containing code that can be imported and reused. They help organize code into logical units, prevent namespace pollution, and enable code reuse across projects. A module is simply a .py file that can contain functions, classes, and variables.

Importing Modules

Python provides several ways to import modules and their contents.

Import Statements:

# Import entire module
import math
print(math.sqrt(16))  # 4.0
print(math.pi)        # 3.141592653589793

# Import with alias
import math as m
print(m.sqrt(25))  # 5.0

# Import specific items
from math import sqrt, pi
print(sqrt(36))  # 6.0
print(pi)        # 3.14159...

# Import multiple items
from math import sin, cos, tan

# Import with alias (specific items)
from math import sqrt as square_root
print(square_root(49))  # 7.0

# Import everything (not recommended)
from math import *
# Now all math functions are available directly
# But this pollutes the namespace

# Common module aliases (conventions)
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# Import user-defined module
import my_module
from my_module import my_function

Lines 2-5: Import the whole module and access items with dot notation.

Lines 8-10: Use aliases to shorten module names.

Lines 13-16: Import specific functions or variables from a module.

Lines 24-25: Avoid from module import * - it pollutes your namespace.

Lines 28-31: Some modules have conventional aliases everyone uses.

Lines 34-35: Import your own Python files as modules.

Creating Your Own Modules

Any Python file can be a module. Let's create a simple module structure.

Module Example (mymath.py):

# mymath.py - A custom math utilities module

"""Mathematical utility functions."""

PI = 3.14159

def circle_area(radius):
    """Calculate the area of a circle."""
    return PI * radius ** 2

def circle_circumference(radius):
    """Calculate the circumference of a circle."""
    return 2 * PI * radius

def rectangle_area(width, height):
    """Calculate the area of a rectangle."""
    return width * height

# Code that runs only when executed directly
if __name__ == "__main__":
    # This block runs when: python mymath.py
    # It does NOT run when: import mymath
    print("Testing mymath module:")
    print(f"Circle area (r=5): {circle_area(5)}")
    print(f"Circle circumference (r=5): {circle_circumference(5)}")

Using the Module:

# main.py - Using our custom module
import mymath

print(mymath.PI)                    # 3.14159
print(mymath.circle_area(5))        # 78.53975
print(mymath.circle_circumference(3))  # 18.84954

# Import specific functions
from mymath import circle_area, PI

area = circle_area(10)
print(f"Area: {area}")  # Area: 314.159

# Check what's in a module
import mymath
print(dir(mymath))  # Lists all attributes
# ['PI', '__builtins__', '__cached__', ... , 'circle_area', ...]

# Get help on a module
help(mymath)  # Shows docstrings

Lines 5: Module-level variables can be constants.

Lines 8-18: Define functions that will be available when imported.

Lines 21-27: The if __name__ == "__main__" block runs only when the file is executed directly.

Lines 34-35: Access module contents with dot notation.

Lines 44-45: dir() lists all module attributes.

Packages

A package is a collection of modules organized in directories. Packages help structure larger projects and avoid naming conflicts.

Package Structure:

# Directory structure:
# mypackage/
# ├── __init__.py
# ├── core.py
# ├── utils.py
# └── subpackage/
#     ├── __init__.py
#     └── helpers.py

# __init__.py can be empty or contain initialization code
# mypackage/__init__.py
from .core import main_function
from .utils import helper

__version__ = "1.0.0"

# mypackage/core.py
def main_function():
    return "Main function called"

def internal_function():
    return "Internal use only"

# mypackage/utils.py
def helper():
    return "Helper function"

# Using the package
import mypackage
print(mypackage.__version__)  # 1.0.0
print(mypackage.main_function())  # Main function called

# Import from submodule
from mypackage import core
from mypackage.utils import helper

# Import from subpackage
from mypackage.subpackage.helpers import something

Lines 2-8: Package directory structure with __init__.py files.

Lines 11-15: __init__.py can define what the package exports.

Lines 24-26: Import from package uses dot notation.

Lines 29-30: Import specific modules from the package.

💡 __init__.py

The __init__.py file marks a directory as a Python package. It can be empty or contain initialization code. In Python 3.3+, packages can be "namespace packages" without __init__.py, but it's still recommended to include it.

Relative vs Absolute Imports

Within packages, you can use relative imports to import from sibling modules.

Import Types:

# Package structure:
# myapp/
# ├── __init__.py
# ├── models.py
# ├── views.py
# └── utils/
#     ├── __init__.py
#     ├── helpers.py
#     └── validators.py

# Absolute imports (recommended)
# In myapp/views.py:
from myapp.models import User
from myapp.utils.helpers import format_date
from myapp.utils.validators import validate_email

# Relative imports (within package only)
# In myapp/views.py:
from .models import User          # Same level
from .utils.helpers import format_date  # Subdirectory

# In myapp/utils/helpers.py:
from .validators import validate_email  # Same level
from ..models import User         # Parent level (..)

# In myapp/utils/__init__.py:
from .helpers import format_date
from .validators import validate_email

# Relative import syntax:
# .     current package
# ..    parent package
# ...   grandparent package (rare)

# Avoid circular imports!
# If A imports B and B imports A, you get a circular import error
# Solution: Restructure code or import inside functions

Lines 11-13: Absolute imports use the full path from the project root.

Lines 16-20: Relative imports use dots to navigate the package structure.

Lines 23-24: Use .. to import from the parent package.

Lines 32-34: Circular imports happen when modules import each other.

The Python Standard Library

Python comes with a rich standard library. Here are some commonly used modules.

Useful Standard Modules:

# Operating system interface
import os
print(os.getcwd())           # Current directory
os.listdir('.')              # List files
os.environ.get('HOME')       # Environment variables

# Path handling (modern)
from pathlib import Path
home = Path.home()
file = Path("folder/file.txt")

# System-specific parameters
import sys
print(sys.version)           # Python version
print(sys.argv)              # Command line arguments

# Pattern matching
import re
re.findall(r'\d+', "abc123def456")  # ['123', '456']

# Mathematics
import math
math.sqrt(16)    # 4.0
math.ceil(4.3)   # 5

# Random numbers
import random
random.random()           # 0.0 to 1.0
random.randint(1, 10)     # 1 to 10
random.choice([1, 2, 3])  # Random element
random.shuffle(list_)     # Shuffle in place

# Date and time
from datetime import datetime, timedelta
now = datetime.now()
tomorrow = now + timedelta(days=1)

# JSON handling
import json
json.dumps({"a": 1})  # '{"a": 1}'
json.loads('{"a": 1}')  # {"a": 1}

# Collections
from collections import Counter, defaultdict
Counter("hello")  # {'l': 2, 'h': 1, 'e': 1, 'o': 1}

# Itertools
from itertools import chain, combinations
list(chain([1, 2], [3, 4]))  # [1, 2, 3, 4]

# Logging
import logging
logging.basicConfig(level=logging.INFO)
logging.info("Application started")

Lines 2-6: os module for operating system interaction.

Lines 9-11: pathlib provides object-oriented path handling.

Lines 18-19: re module for regular expressions.

Lines 27-31: random module for generating random values.

Lines 34-36: datetime for working with dates and times.

Lines 48-50: logging is better than print for production code.

Installing External Packages with pip

pip is Python's package installer. It downloads and installs packages from the Python Package Index (PyPI).

pip Commands:

# Install a package
pip install requests

# Install specific version
pip install requests==2.28.0

# Install with version constraints
pip install "requests>=2.25.0,<3.0.0"

# List installed packages
pip list

# Show package details
pip show requests

# Uninstall a package
pip uninstall requests

# Create requirements file
pip freeze > requirements.txt

# Install from requirements file
pip install -r requirements.txt

# Upgrade a package
pip install --upgrade requests

# Install in user space (no admin needed)
pip install --user package_name

# Search for packages (command line)
# pip search is disabled; use pypi.org

# Install from git repository
pip install git+https://github.com/user/repo.git

# Install in development mode
pip install -e ./my-local-package

Lines 2-3: Basic package installation from PyPI.

Lines 6-9: Install specific versions or version ranges.

Lines 15-16: View installed packages and their details.

Lines 22-25: Requirements files enable reproducible installations.

Lines 34-35: Install from source code repositories.

Virtual Environments

Virtual environments create isolated Python installations, preventing package conflicts between projects.

Working with Virtual Environments:

# Create virtual environment
python -m venv myenv

# Activate (Linux/macOS)
source myenv/bin/activate

# Activate (Windows)
myenv\Scripts\activate

# Deactivate
deactivate

# Once activated, pip installs to the virtual environment
pip install requests  # Only in this environment

# Create with specific Python version
python3.11 -m venv myenv

# Virtual environment structure:
# myenv/
# ├── bin/ (Linux/macOS) or Scripts/ (Windows)
# │   ├── activate
# │   ├── python
# │   └── pip
# ├── lib/
# │   └── python3.x/
# │       └── site-packages/
# └── pyvenv.cfg

# Using virtualenv (third-party, more features)
pip install virtualenv
virtualenv myenv

# Using poetry (modern dependency management)
pip install poetry
poetry new myproject
poetry add requests  # Add dependency

# Using conda (Anaconda/Miniconda)
conda create -n myenv python=3.11
conda activate myenv
conda install requests

Lines 2-3: Create a virtual environment with Python's built-in venv module.

Lines 5-9: Activate/deactivate the virtual environment.

Lines 13-14: When activated, all pip commands affect only this environment.

Lines 17-18: Specify which Python version to use.

Lines 32-34: Poetry is a modern tool for dependency management.

Lines 37-39: Conda is popular in data science environments.

💡 Always Use Virtual Environments

Never install project dependencies globally. Each project should have its own virtual environment. This prevents version conflicts and makes projects reproducible.

requirements.txt

The requirements.txt file specifies project dependencies for reproducible installations.

Requirements File:

# requirements.txt
# Basic format
requests>=2.28.0
flask>=2.0.0
pandas>=1.5.0

# With exact versions (more deterministic)
requests==2.28.1
flask==2.2.2
pandas==1.5.2

# With comments
numpy>=1.21.0  # Required for data processing
matplotlib>=3.5.0  # For visualizations

# Environment markers
pywin32>=300; sys_platform == 'win32'

# Direct URLs
git+https://github.com/user/repo.git@main#egg=package

# Install from requirements file
pip install -r requirements.txt

# Generate requirements (freeze current environment)
pip freeze > requirements.txt

# Development requirements (requirements-dev.txt)
pytest>=7.0.0
black>=22.0.0
flake8>=5.0.0

# Install development requirements
pip install -r requirements-dev.txt

Lines 3-6: Use >= to allow compatible updates.

Lines 9-11: Use == for exact versions (most deterministic).

Lines 18-19: Environment markers for platform-specific dependencies.

Lines 24-27: Generate requirements from installed packages.

Lines 30-35: Separate development dependencies into their own file.
