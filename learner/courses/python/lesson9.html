Lesson 9: String Manipulation

Master string methods, formatting, and text processing techniques

Strings in Depth

Strings are sequences of characters and one of Python's most commonly used data types. Python provides rich functionality for creating, formatting, and manipulating text. Understanding strings deeply will help you work with user input, files, APIs, and more.

String Creation and Basics

Strings can be created with single, double, or triple quotes. Triple quotes allow multi-line strings.

Creating Strings:

# Single and double quotes (identical behavior)
single = 'Hello'
double = "Hello"
print(single == double)  # True

# Escape characters
escaped = "He said \"Hello\""
print(escaped)  # He said "Hello"
path = "C:\\Users\\Documents"  # Backslash escape
line_break = "Line 1\nLine 2"  # Newline

# Raw strings (ignore escapes)
raw_path = r"C:\Users\Documents"
print(raw_path)  # C:\Users\Documents

# Multi-line strings
multi = """
This is a
multi-line string
"""

# Triple quotes preserve formatting
html = """


    
Title



"""

# String length
text = "Python"
print(len(text))  # 6

# Concatenation
full = "Hello" + " " + "World"
repeated = "Hi" * 3  # "HiHiHi"

Lines 2-4: Single and double quotes work identically in Python.

Lines 7-10: Use backslash for escape sequences: \", \\, \n, \t.

Lines 13-14: Raw strings (r"...") treat backslashes literally.

Lines 17-26: Triple quotes create multi-line strings preserving whitespace.

Lines 29-33: Strings support len(), +, and * operations.

String Indexing and Slicing

Strings are sequences, so you can access individual characters or extract substrings using indexing and slicing.

Accessing String Parts:

text = "Python"

# Indexing
print(text[0])    # 'P' (first character)
print(text[-1])   # 'n' (last character)
print(text[-2])   # 'o' (second to last)

# Slicing: [start:end:step]
print(text[0:3])  # 'Pyt' (characters 0, 1, 2)
print(text[:3])   # 'Pyt' (from start to index 2)
print(text[2:])   # 'thon' (from index 2 to end)
print(text[::2])  # 'Pto' (every second character)

# Reverse a string
print(text[::-1])  # 'nohtyP'

# Get last 3 characters
print(text[-3:])   # 'hon'

# Strings are immutable
# text[0] = 'J'  # TypeError!

# To "modify", create new string
new_text = 'J' + text[1:]
print(new_text)  # 'Jython'

Lines 5-7: Positive index from start, negative from end (-1 is last).

Lines 10-13: Slice syntax [start:end:step]. End is exclusive.

Line 16: [::-1] is the Pythonic way to reverse a string.

Lines 22-23: Strings are immutable - create new strings instead of modifying.

String Methods

Python strings have many built-in methods for transformation, searching, and testing.

Case Methods:

text = "Hello World"

# Case transformations
print(text.upper())      # "HELLO WORLD"
print(text.lower())      # "hello world"
print(text.title())      # "Hello World"
print(text.capitalize()) # "Hello world"
print(text.swapcase())   # "hELLO wORLD"

# Whitespace handling
messy = "  Hello World  "
print(messy.strip())     # "Hello World" (both ends)
print(messy.lstrip())    # "Hello World  " (left only)
print(messy.rstrip())    # "  Hello World" (right only)

# Strip specific characters
chars = "xxHelloxx"
print(chars.strip('x'))  # "Hello"

# Center, left, right align
print("Hi".center(10))      # "    Hi    "
print("Hi".ljust(10, '-'))  # "Hi--------"
print("Hi".rjust(10, '-'))  # "--------Hi"

Lines 5-9: Various case conversion methods.

Lines 12-17: strip(), lstrip(), rstrip() remove whitespace.

Lines 20-21: Can specify characters to strip instead of whitespace.

Lines 24-26: Alignment methods for text formatting.

Search and Replace:

text = "Hello World World"

# Find substring position
print(text.find("World"))     # 6 (first occurrence)
print(text.find("Python"))    # -1 (not found)
print(text.rfind("World"))    # 12 (last occurrence)

# Find with index (raises error if not found)
print(text.index("World"))    # 6
# text.index("Python")        # ValueError

# Count occurrences
print(text.count("World"))    # 2
print(text.count("o"))        # 3

# Replace
print(text.replace("World", "Python"))
# "Hello Python Python"

# Replace with count limit
print(text.replace("World", "Python", 1))
# "Hello Python World"

# Check start/end
print(text.startswith("Hello"))  # True
print(text.endswith("World"))    # True

Lines 5-7: find() returns -1 if not found, rfind() searches from end.

Lines 10-11: index() is like find but raises ValueError if not found.

Lines 14-15: count() returns the number of occurrences.

Lines 18-22: replace() can limit number of replacements.

Lines 25-26: Check what string starts or ends with.

Splitting and Joining:

# Split string into list
sentence = "Hello World Python"
words = sentence.split()  # Default: whitespace
print(words)  # ['Hello', 'World', 'Python']

# Split with delimiter
csv = "apple,banana,cherry"
fruits = csv.split(',')
print(fruits)  # ['apple', 'banana', 'cherry']

# Limit splits
data = "a:b:c:d"
print(data.split(':', 2))  # ['a', 'b', 'c:d']

# Partition (splits into 3 parts)
text = "user@example.com"
before, sep, after = text.partition('@')
print(before, after)  # user example.com

# Join list into string
words = ['Hello', 'World']
print(' '.join(words))   # "Hello World"
print('-'.join(words))   # "Hello-World"
print(''.join(words))    # "HelloWorld"

# Join with path separator
import os
paths = ['folder', 'subfolder', 'file.txt']
full_path = os.path.join(*paths)

# Split lines
multi = "Line 1\nLine 2\nLine 3"
lines = multi.splitlines()
print(lines)  # ['Line 1', 'Line 2', 'Line 3']

Lines 3-5: split() without argument splits on any whitespace.

Lines 8-10: Specify delimiter for custom splitting.

Lines 17-19: partition() always returns 3 parts: before, separator, after.

Lines 22-25: join() is the reverse of split - combines list elements.

Lines 33-35: splitlines() handles different line ending styles.

String Testing Methods

Python provides methods to test string content, useful for validation.

Validation Methods:

# Character type tests
print("hello".isalpha())     # True (letters only)
print("hello123".isalnum())  # True (letters and numbers)
print("12345".isdigit())     # True (digits only)
print("123.45".isnumeric())  # False (not purely numeric)
print("   ".isspace())       # True (whitespace only)

# Case tests
print("hello".islower())     # True
print("HELLO".isupper())     # True
print("Hello World".istitle())  # True

# Check if printable
print("Hello!".isprintable())  # True
print("Hello\n".isprintable()) # False

# Practical validation
def validate_username(username):
    if not username:
        return False, "Username cannot be empty"
    if not username.isalnum():
        return False, "Username must be alphanumeric"
    if len(username) < 3:
        return False, "Username too short"
    return True, "Valid username"

print(validate_username("user123"))  # (True, "Valid username")

# Check for valid identifier
print("my_var".isidentifier())   # True
print("123var".isidentifier())   # False (starts with number)

Lines 3-7: Test what characters the string contains.

Lines 10-12: Check case of string content.

Lines 19-27: Combining tests for input validation.

Lines 30-31: isidentifier() checks if string is a valid Python identifier.

String Formatting

Python offers multiple ways to format strings with dynamic values. F-strings are the modern, recommended approach.

F-Strings (Python 3.6+):

name = "Alice"
age = 25
score = 95.6789

# Basic f-string
print(f"Hello, {name}!")

# Expressions inside braces
print(f"Next year you'll be {age + 1}")

# Format specifications
print(f"Score: {score:.2f}")      # "Score: 95.68"
print(f"Score: {score:.0f}")      # "Score: 96" (no decimals)
print(f"Number: {42:05d}")        # "Number: 00042"
print(f"Percent: {0.856:.1%}")    # "Percent: 85.6%"

# Alignment and width
print(f"|{name:<10}|")  # |Alice     | (left)
print(f"|{name:>10}|")  # |     Alice| (right)
print(f"|{name:^10}|")  # |  Alice   | (center)

# Number formatting
big_num = 1234567.89
print(f"{big_num:,.2f}")   # "1,234,567.89"
print(f"{big_num:_,.2f}")  # "1_234_567.89" (underscore separator)

# Date formatting
from datetime import datetime
now = datetime.now()
print(f"Date: {now:%Y-%m-%d %H:%M}")

# Debug mode (Python 3.8+)
x = 10
y = 20
print(f"{x=}, {y=}")  # "x=10, y=20"
print(f"{x + y =}")   # "x + y =30"

Lines 7-8: F-strings evaluate expressions inside curly braces.

Lines 11-15: Format specifiers after colon: .Nf for decimals, :Nd for width.

Lines 18-20: Alignment: < for left, > for right, ^ for center.

Lines 23-25: Number formatting with comma or underscore separators.

Lines 34-36: Debug mode shows variable name and value.

Alternative Formatting Methods:

name = "Alice"
age = 25

# format() method
print("Hello, {}".format(name))
print("Hello, {name}, age {age}".format(name=name, age=age))

# Positional and named arguments
print("{0} is {1} years old. {0} likes Python.".format(name, age))

# %-formatting (old style)
print("Hello, %s" % name)
print("Age: %d, Score: %.2f" % (age, 95.67))

# Template strings (for user-provided format strings)
from string import Template
t = Template("Hello, $name!")
print(t.substitute(name="Alice"))

# Safe substitution (doesn't fail on missing)
t = Template("Hello, $name, your score is $score")
result = t.safe_substitute(name="Alice")
print(result)  # "Hello, Alice, your score is $score"

Lines 5-6: format() method is flexible but more verbose than f-strings.

Line 9: Can reference arguments by position multiple times.

Lines 12-13: %-formatting is older but still seen in legacy code.

Lines 16-22: Template strings are safer for user-provided format strings.

Regular Expressions

For complex pattern matching, Python's re module provides regular expression support.

Pattern Matching:

import re

text = "My phone is 555-1234 and email is user@example.com"

# Search for pattern
match = re.search(r'\d{3}-\d{4}', text)
if match:
    print(match.group())  # "555-1234"
    print(match.start())  # 12 (start position)

# Find all matches
emails = re.findall(r'\w+@\w+\.\w+', text)
print(emails)  # ['user@example.com']

# Find all with groups
text = "John: 25, Jane: 30, Bob: 35"
matches = re.findall(r'(\w+): (\d+)', text)
print(matches)  # [('John', '25'), ('Jane', '30'), ('Bob', '35')]

# Replace with pattern
result = re.sub(r'\d{3}-\d{4}', 'XXX-XXXX', text)
print(result)  # Phone number replaced

# Split by pattern
words = re.split(r'[,\s]+', "apple, banana, cherry")
print(words)  # ['apple', 'banana', 'cherry']

# Common patterns
patterns = {
    'email': r'^[\w\.-]+@[\w\.-]+\.\w+$',
    'phone': r'^\d{3}-\d{3}-\d{4}$',
    'url': r'^https?://[\w\.-]+\.\w+',
    'date': r'^\d{4}-\d{2}-\d{2}$'
}

def validate(pattern_name, text):
    return bool(re.match(patterns[pattern_name], text))

print(validate('email', 'user@example.com'))  # True

Lines 5-8: re.search() finds first match, returns a Match object.

Lines 11-12: re.findall() returns all matches as a list.

Lines 15-17: Groups with parentheses return tuples.

Lines 20-21: re.sub() replaces pattern matches.

Lines 24-35: Common regex patterns for validation.

💡 Regex Tip

Use raw strings (r"...") for regex patterns to avoid escaping backslashes. Regular expressions are powerful but complex - start simple and test incrementally.
