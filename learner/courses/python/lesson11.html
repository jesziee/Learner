Lesson 11: Object-Oriented Programming

Understand classes, objects, inheritance, and encapsulation

What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that organizes code into objects containing data (attributes) and behavior (methods). Python is a multi-paradigm language that fully supports OOP. OOP helps create modular, reusable, and maintainable code by modeling real-world entities as objects.

Classes and Objects

A class is a blueprint for creating objects. An object is an instance of a class with specific attribute values.

Creating Classes:

# Basic class definition
class Dog:
    """A simple Dog class."""
    
    # Class attribute (shared by all instances)
    species = "Canis familiaris"
    
    # Constructor (initializer)
    def __init__(self, name, age):
        # Instance attributes (unique to each instance)
        self.name = name
        self.age = age
    
    # Instance method
    def bark(self):
        return f"{self.name} says Woof!"
    
    # Method with parameter
    def birthday(self):
        self.age += 1
        return f"{self.name} is now {age}!"

# Create objects (instances)
dog1 = Dog("Buddy", 3)
dog2 = Dog("Max", 5)

# Access attributes
print(dog1.name)        # Buddy
print(dog2.age)         # 5
print(dog1.species)     # Canis familiaris

# Call methods
print(dog1.bark())      # Buddy says Woof!

# Modify attributes
dog1.age = 4
print(dog1.age)         # 4

# Check class and instance
print(type(dog1))       # 
print(isinstance(dog1, Dog))  # True

Lines 2-4: Use class keyword. Docstrings describe the class.

Lines 7-8: Class attributes are shared by all instances of the class.

Lines 11-14: __init__ is the constructor. self refers to the instance being created.

Lines 17-18: Instance methods take self as the first parameter.

Lines 24-25: Create objects by calling the class like a function.

Lines 28-30: Access attributes using dot notation.

💡 Understanding 'self'

self is a convention (not a keyword) that refers to the current instance. It's automatically passed as the first argument to instance methods. You can name it differently, but self is the standard convention.

Class Attributes vs Instance Attributes

Understanding the difference between class and instance attributes is crucial for proper OOP design.

Attribute Types:

class Counter:
    # Class attribute - shared by all instances
    total_count = 0
    
    def __init__(self, name):
        # Instance attribute - unique to each instance
        self.name = name
        self.count = 0
        # Modify class attribute
        Counter.total_count += 1
    
    def increment(self):
        self.count += 1
    
    @classmethod
    def get_total(cls):
        """Class method - works with class, not instance."""
        return cls.total_count

# Create instances
c1 = Counter("First")
c2 = Counter("Second")

print(Counter.total_count)  # 2 (class attribute)
print(c1.count)             # 0 (instance attribute)
print(c2.count)             # 0 (instance attribute)

c1.increment()
print(c1.count)             # 1
print(c2.count)             # 0 (unchanged)

# Be careful with mutable class attributes!
class Buggy:
    items = []  # Shared by all instances!

a = Buggy()
b = Buggy()
a.items.append("x")
print(b.items)  # ['x'] - Not what you might expect!

# Better approach
class Fixed:
    def __init__(self):
        self.items = []  # Each instance has its own list

Lines 3-4: Class attributes are defined outside methods and shared.

Lines 7-10: Instance attributes are defined in __init__ with self.

Lines 15-17: @classmethod creates methods that work with the class.

Lines 28-31: Modifying one instance doesn't affect others' instance attributes.

Lines 34-42: Mutable class attributes can cause unexpected sharing - avoid them!

Special Methods (Dunder Methods)

Special methods with double underscores (dunder methods) let you define how objects behave with built-in operations.

Common Special Methods:

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    # String representation (for users)
    def __str__(self):
        return f"Vector({self.x}, {self.y})"
    
    # Representation (for developers)
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    
    # Addition: v1 + v2
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)
    
    # Subtraction: v1 - v2
    def __sub__(self, other):
        return Vector(self.x - other.x, self.y - other.y)
    
    # Multiplication: v * scalar
    def __mul__(self, scalar):
        return Vector(self.x * scalar, self.y * scalar)
    
    # Length: len(v)
    def __len__(self):
        return int((self.x**2 + self.y**2) ** 0.5)
    
    # Equality: v1 == v2
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y
    
    # Hash for use in sets and dicts
    def __hash__(self):
        return hash((self.x, self.y))
    
    # Boolean conversion: bool(v)
    def __bool__(self):
        return self.x != 0 or self.y != 0

# Usage
v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(v1)           # Vector(3, 4) - uses __str__
print(v1 + v2)      # Vector(4, 6)
print(v1 * 2)       # Vector(6, 8)
print(len(v1))      # 5 (magnitude)
print(v1 == v2)     # False

Lines 8-9: __str__ defines how the object appears in print().

Lines 12-13: __repr__ should return a valid Python expression.

Lines 16-18: __add__ enables the + operator.

Lines 25-27: __mul__ enables multiplication.

Lines 36-37: __eq__ defines equality comparison.

Lines 48-53: These special methods make custom objects work naturally with Python syntax.

Inheritance

Inheritance allows a class to inherit attributes and methods from another class. This promotes code reuse and establishes relationships between classes.

Class Inheritance:

# Base class (parent)
class Animal:
    def __init__(self, name):
        self.name = name
    
    def speak(self):
        raise NotImplementedError("Subclass must implement")
    
    def info(self):
        return f"I am {self.name}"

# Derived class (child)
class Dog(Animal):
    def __init__(self, name, breed):
        # Call parent constructor
        super().__init__(name)
        self.breed = breed
    
    # Override parent method
    def speak(self):
        return f"{self.name} says Woof!"
    
    # Add new method
    def fetch(self):
        return f"{self.name} fetches the ball!"

class Cat(Animal):
    def speak(self):
        return f"{self.name} says Meow!"

# Create instances
dog = Dog("Buddy", "Golden Retriever")
cat = Cat("Whiskers")

print(dog.speak())      # Buddy says Woof!
print(cat.speak())      # Whiskers says Meow!
print(dog.info())       # I am Buddy (inherited)
print(dog.breed)        # Golden Retriever

# Check inheritance
print(isinstance(dog, Dog))     # True
print(isinstance(dog, Animal))  # True
print(issubclass(Dog, Animal))  # True

# Multiple inheritance
class Flying:
    def fly(self):
        return "I can fly!"

class Swimming:
    def swim(self):
        return "I can swim!"

class Duck(Animal, Flying, Swimming):
    def speak(self):
        return f"{self.name} says Quack!"

donald = Duck("Donald")
print(donald.fly())   # I can fly!
print(donald.swim())  # I can swim!

Lines 3-10: Base class defines common interface. Abstract methods raise NotImplementedError.

Lines 13-22: Child class inherits from parent (in parentheses). Use super() to call parent methods.

Lines 17-18: Override parent methods by redefining them.

Lines 37-39: isinstance() and issubclass() check inheritance relationships.

Lines 42-57: Python supports multiple inheritance - class inherits from multiple parents.

Encapsulation and Access Control

Encapsulation hides internal details and protects data. Python uses conventions rather than strict access control.

Access Control:

class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner           # Public
        self._balance = balance      # Protected (convention)
        self.__id = id(self)         # Name mangled (private-ish)
    
    # Public method
    def deposit(self, amount):
        if amount > 0:
            self._balance += amount
            return True
        return False
    
    def withdraw(self, amount):
        if 0 < amount <= self._balance:
            self._balance -= amount
            return True
        return False
    
    # Property for controlled access
    @property
    def balance(self):
        """Read-only access to balance."""
        return self._balance
    
    # Setter with validation
    @balance.setter
    def balance(self, value):
        if value >= 0:
            self._balance = value
        else:
            raise ValueError("Balance cannot be negative")

# Usage
account = BankAccount("Alice", 1000)

print(account.owner)       # Alice (public)
print(account.balance)     # 1000 (via property)

# Can access "protected" (but shouldn't)
print(account._balance)    # 1000 (accessible but discouraged)

# Name mangled attribute
# print(account.__id)      # AttributeError
print(account._BankAccount__id)  # Accessible but very discouraged

# Use methods for operations
account.deposit(500)
print(account.balance)     # 1500

Lines 4-6: Public (no prefix), protected (_), private (__).

Lines 9-19: Public methods provide controlled access to internal data.

Lines 22-24: @property creates read-only attributes.

Lines 27-31: Property setters allow validation on assignment.

Lines 38-39: Protected attributes can be accessed but convention says don't.

Lines 42-43: Name mangling makes private attributes harder to access.

💡 Python's Philosophy

Python trusts developers. Unlike other languages, it doesn't enforce strict access control. The convention is: use a single underscore for "internal use" and double underscore for name mangling. True privacy doesn't exist in Python!

Class Methods and Static Methods

Besides instance methods, Python supports class methods (that receive the class) and static methods (that receive neither class nor instance).

Method Types:

class Person:
    population = 0
    
    def __init__(self, name):
        self.name = name
        Person.population += 1
    
    # Instance method - receives self
    def greet(self):
        return f"Hello, I'm {self.name}"
    
    # Class method - receives cls
    @classmethod
    def from_string(cls, string):
        """Alternative constructor from string."""
        name = string.split("-")[0]
        return cls(name)
    
    @classmethod
    def get_population(cls):
        return f"Population: {cls.population}"
    
    # Static method - receives nothing
    @staticmethod
    def is_valid_name(name):
        """Utility function related to Person."""
        return len(name) >= 2 and name.isalpha()

# Instance method
p1 = Person("Alice")
print(p1.greet())  # Hello, I'm Alice

# Class method as alternative constructor
p2 = Person.from_string("Bob-Smith")
print(p2.name)     # Bob

# Class method on class or instance
print(Person.get_population())  # Population: 2
print(p1.get_population())      # Also works

# Static method - no self or cls needed
print(Person.is_valid_name("Alice"))  # True
print(Person.is_valid_name("A"))      # False

Lines 9-10: Instance methods take self and can access instance attributes.

Lines 13-18: @classmethod methods take cls and can access class attributes.

Lines 15-17: Class methods are often used as alternative constructors.

Lines 24-26: @staticmethod methods don't take self or cls.

Lines 32-33: Static methods are utility functions logically related to the class.

Data Classes

Python 3.7+ introduced data classes, which automatically generate common methods like __init__, __repr__, and __eq__.

Data Classes:

from dataclasses import dataclass, field
from typing import List

@dataclass
class Product:
    name: str
    price: float
    quantity: int = 1
    tags: List[str] = field(default_factory=list)
    
    def total_cost(self) -> float:
        return self.price * self.quantity

# Create instances
p1 = Product("Widget", 9.99)
p2 = Product("Gadget", 19.99, quantity=3)

print(p1)  # Product(name='Widget', price=9.99, quantity=1, tags=[])

# Auto-generated __eq__
print(p1 == Product("Widget", 9.99))  # True

# Access methods
print(p2.total_cost())  # 59.97

# Frozen dataclass (immutable)
@dataclass(frozen=True)
class Point:
    x: float
    y: float

p = Point(3.0, 4.0)
# p.x = 5.0  # FrozenInstanceError

# Post-init processing
@dataclass
class Rectangle:
    width: float
    height: float
    area: float = field(init=False)
    
    def __post_init__(self):
        self.area = self.width * self.height

r = Rectangle(5, 3)
print(r.area)  # 15.0

Lines 5-8: Type annotations define attributes. Default values are supported.

Line 8: Use field(default_factory=list) for mutable defaults.

Lines 10-11: Data classes can have regular methods too.

Lines 22-26: frozen=True makes the dataclass immutable.

Lines 29-37: __post_init__ runs after initialization for computed fields.
