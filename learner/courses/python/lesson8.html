Lesson 8: Dictionaries & Sets

Use key-value pairs and unique collections with dictionaries and sets

What are Dictionaries?

Dictionaries are collections of key-value pairs. Each key maps to a value, allowing you to store and retrieve data using meaningful names instead of numeric indices. Keys must be unique and immutable (like strings, numbers, or tuples).

Creating Dictionaries

Create dictionaries using curly braces {} with key:value pairs, or the dict() constructor.

Dictionary Creation:

# Empty dictionary
empty = {}
also_empty = dict()

# Dictionary with initial values
person = {
    "name": "Alice",
    "age": 25,
    "city": "New York"
}

# Using dict() constructor
person2 = dict(name="Bob", age=30, city="LA")

# From list of tuples
items = [("a", 1), ("b", 2)]
d = dict(items)  # {'a': 1, 'b': 2}

# Dictionary with different key types
mixed_keys = {
    "string": "value",
    42: "number key",
    (1, 2): "tuple key"
}

# Nested dictionaries
company = {
    "name": "TechCorp",
    "employees": {
        "count": 50,
        "departments": ["Engineering", "Sales", "HR"]
    }
}

# Dictionary comprehension
squares = {x: x**2 for x in range(1, 6)}
print(squares)  # {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

Lines 2-3: Two ways to create empty dictionaries.

Lines 6-10: Curly braces with key:value pairs is the most common syntax.

Lines 13-14: The dict() constructor can take keyword arguments.

Lines 17-22: Keys can be any immutable type: strings, numbers, tuples.

Lines 25-30: Dictionaries can contain other dictionaries as values.

Lines 33-34: Dictionary comprehension creates dictionaries from iterables.

Accessing Dictionary Values

Access values using keys in square brackets or the get() method for safer access.

Value Access:

person = {"name": "Alice", "age": 25, "city": "NYC"}

# Access by key
print(person["name"])  # Alice
print(person["age"])   # 25

# KeyError if key doesn't exist
# print(person["email"])  # KeyError!

# Safe access with get()
print(person.get("name"))      # Alice
print(person.get("email"))     # None (no error)
print(person.get("email", "N/A"))  # N/A (default value)

# Check if key exists
print("name" in person)   # True
print("email" in person)  # False

# Access nested values
company = {
    "name": "TechCorp",
    "employees": {"count": 50}
}
print(company["employees"]["count"])  # 50

# Using .keys(), .values(), .items()
print(person.keys())    # dict_keys(['name', 'age', 'city'])
print(person.values())  # dict_values(['Alice', 25, 'NYC'])
print(person.items())   # dict_items([('name', 'Alice'), ...])

# Convert to lists
keys_list = list(person.keys())
values_list = list(person.values())

Lines 4-5: Use square brackets with key name to access values.

Line 8: Missing keys cause KeyError when using bracket notation.

Lines 11-13: get() returns None (or a default) for missing keys.

Lines 16-17: Use in to check if a key exists.

Lines 24-25: Access nested dictionary values with chained brackets.

Lines 28-33: View methods return all keys, values, or key-value pairs.

Modifying Dictionaries

Dictionaries are mutable. You can add, update, and delete key-value pairs.

Dictionary Modification:

person = {"name": "Alice", "age": 25}

# Add or update by assignment
person["city"] = "NYC"      # Add new key
person["age"] = 26          # Update existing
print(person)  # {'name': 'Alice', 'age': 26, 'city': 'NYC'}

# Update multiple pairs
person.update({"email": "alice@example.com", "age": 27})
print(person)

# Update from keyword arguments
person.update(country="USA", phone="555-1234")

# Remove items
removed = person.pop("phone")  # Remove and return value
print(removed)  # 555-1234

# Pop with default (avoids KeyError)
email = person.pop("email", "no email")

# Remove last inserted (Python 3.7+)
last = person.popitem()
print(last)  # ('city', 'NYC')

# Delete key
del person["age"]

# Clear all
person.clear()
print(person)  # {}

# Set default - add if doesn't exist
counts = {}
counts.setdefault("a", 0)  # Returns 0, sets counts["a"] = 0
counts["a"] += 1

Lines 4-6: Assign to a key to add new or update existing entries.

Lines 9-12: update() merges another dictionary or keyword args.

Lines 15-16: pop() removes a key and returns its value.

Lines 19-20: pop() with default won't raise KeyError for missing keys.

Lines 23-24: popitem() removes and returns the last item (LIFO).

Lines 30-32: setdefault() sets a value only if the key doesn't exist.

Iterating Dictionaries

Loop through dictionaries using various techniques depending on whether you need keys, values, or both.

Dictionary Iteration:

person = {"name": "Alice", "age": 25, "city": "NYC"}

# Iterate over keys (default)
for key in person:
    print(key)

# Explicitly iterate keys
for key in person.keys():
    print(key)

# Iterate over values
for value in person.values():
    print(value)

# Iterate over key-value pairs
for key, value in person.items():
    print(f"{key}: {value}")

# Get key and value with index
for i, (key, value) in enumerate(person.items()):
    print(f"{i}. {key} = {value}")

# Build dictionary from two lists
keys = ["name", "age", "city"]
values = ["Bob", 30, "LA"]
person = dict(zip(keys, values))
print(person)  # {'name': 'Bob', 'age': 30, 'city': 'LA'}

# Filter dictionary
scores = {"Alice": 85, "Bob": 72, "Charlie": 90}
passed = {k: v for k, v in scores.items() if v >= 75}
print(passed)  # {'Alice': 85, 'Charlie': 90}

Lines 4-7: Iterating directly over a dict gives keys.

Lines 13-14: .values() lets you iterate over values only.

Lines 17-18: .items() gives key-value pairs as tuples.

Lines 21-22: Unpacking pairs in the loop header is clean and readable.

Lines 31-33: Dictionary comprehension can filter entries.

Introduction to Sets

Sets are unordered collections of unique elements. They're useful for membership testing, removing duplicates, and mathematical operations like union and intersection.

Creating Sets:

# Empty set (note: {} creates an empty dict!)
empty = set()

# Set with values
fruits = {"apple", "banana", "cherry"}
numbers = set([1, 2, 2, 3, 3, 3])  # Deduplicates
print(numbers)  # {1, 2, 3}

# Set from string
letters = set("hello")
print(letters)  # {'h', 'e', 'l', 'o'} - order varies

# Frozenset (immutable set)
frozen = frozenset([1, 2, 3])
# frozen.add(4)  # Error - can't modify

# Set comprehension
squares = {x**2 for x in range(5)}
print(squares)  # {0, 1, 4, 9, 16}

# Sets can only contain immutable (hashable) types
valid = {1, "hello", (1, 2)}
# invalid = {[1, 2]}  # TypeError - lists aren't hashable

Line 2: Use set() for empty set. {} creates an empty dictionary!

Lines 5-7: Sets automatically remove duplicates.

Lines 10-11: Sets from iterables keep only unique elements.

Lines 14-16: frozenset is an immutable, hashable set.

Lines 22-23: Sets can only contain hashable (immutable) elements.

Set Operations

Sets support mathematical operations like union, intersection, difference, and more. These are powerful tools for comparing collections.

Mathematical Set Operations:

a = {1, 2, 3, 4, 5}
b = {4, 5, 6, 7, 8}

# Union (all elements from both)
print(a | b)        # {1, 2, 3, 4, 5, 6, 7, 8}
print(a.union(b))   # Same result

# Intersection (elements in both)
print(a & b)             # {4, 5}
print(a.intersection(b)) # Same result

# Difference (in a but not in b)
print(a - b)          # {1, 2, 3}
print(a.difference(b))# Same result

# Symmetric difference (in either, but not both)
print(a ^ b)                   # {1, 2, 3, 6, 7, 8}
print(a.symmetric_difference(b)) # Same result

# Subset and superset tests
c = {1, 2}
print(c.issubset(a))     # True
print(a.issuperset(c))   # True
print(c <= a)            # True (subset operator)
print(a >= c)            # True (superset operator)

# Disjoint (no common elements)
d = {10, 11}
print(a.isdisjoint(d))  # True

# Membership testing (very fast!)
print(3 in a)  # True
print(10 in a) # False

Lines 5-6: Union (|) combines all unique elements from both sets.

Lines 9-10: Intersection (&) keeps only elements in both sets.

Lines 13-14: Difference (-) keeps elements in left set but not right.

Lines 17-18: Symmetric difference (^) keeps elements in either but not both.

Lines 21-25: Test subset/superset relationships.

Lines 31-32: Sets provide O(1) membership testing - much faster than lists!

Modifying Sets

Sets are mutable and support various methods for adding and removing elements.

Set Modification:

fruits = {"apple", "banana"}

# Add single element
fruits.add("cherry")
print(fruits)  # {'apple', 'banana', 'cherry'}

# Add multiple elements
fruits.update(["date", "elderberry"])
fruits.update({"fig", "grape"})  # Also works with sets

# Remove element (raises KeyError if missing)
fruits.remove("apple")

# Discard element (no error if missing)
fruits.discard("mango")  # No error even though not in set

# Pop random element
item = fruits.pop()
print(f"Popped: {item}")

# Clear all elements
fruits.clear()

# Update operations (modify in place)
a = {1, 2, 3}
b = {3, 4, 5}

a.update(b)         # a becomes {1, 2, 3, 4, 5}
a.intersection_update({2, 3, 4})  # a becomes {2, 3, 4}
a.difference_update({2})  # a becomes {3, 4}

Lines 5-6: add() inserts a single element.

Lines 9-10: update() adds multiple elements from any iterable.

Lines 13-14: remove() raises KeyError if element doesn't exist.

Lines 17-18: discard() silently ignores missing elements.

Lines 21-22: pop() removes an arbitrary element (sets are unordered).

Lines 28-32: In-place versions of set operations.

Use Cases for Dictionaries and Sets

Here are practical examples showing when to use dictionaries and sets effectively.

Practical Examples:

# Counting with dictionary
text = "the quick brown fox jumps over the lazy dog"
word_count = {}

for word in text.split():
    word_count[word] = word_count.get(word, 0) + 1

print(word_count["the"])  # 2

# Using Counter from collections
from collections import Counter
counts = Counter(text.split())
print(counts.most_common(2))  # [('the', 2), ...]

# Grouping with dictionary
from collections import defaultdict
students = [("Alice", "A"), ("Bob", "B"), ("Charlie", "A")]

groups = defaultdict(list)
for name, grade in students:
    groups[grade].append(name)

print(groups["A"])  # ['Alice', 'Charlie']

# Remove duplicates while preserving order (Python 3.7+)
items = [1, 2, 2, 3, 3, 3, 4]
unique = list(dict.fromkeys(items))
print(unique)  # [1, 2, 3, 4]

# Find common elements
list1 = [1, 2, 3, 4, 5]
list2 = [4, 5, 6, 7, 8]
common = list(set(list1) & set(list2))
print(common)  # [4, 5]

# Cache/memoization with dictionary
def fibonacci(n, cache={}):
    if n in cache:
        return cache[n]
    if n <= 1:
        return n
    cache[n] = fibonacci(n-1) + fibonacci(n-2)
    return cache[n]

print(fibonacci(50))  # Fast due to caching!

Lines 4-9: Manual counting pattern with dictionary.

Lines 12-14: Counter is a specialized dictionary for counting.

Lines 17-22: defaultdict automatically creates default values for missing keys.

Lines 25-27: Deduplicate while preserving order using dict.

Lines 30-32: Set operations for finding common elements.

Lines 35-42: Dictionary as cache for memoization.
