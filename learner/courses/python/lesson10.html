Lesson 10: File Handling

Read from and write to files, handle different file formats

Working with Files

File handling is essential for most real-world applications. Python makes it easy to read, write, and manipulate files of various types. Understanding file operations allows you to save data, process logs, work with configurations, and more.

Opening Files

Use the open() function to open files. The with statement ensures files are properly closed, even if errors occur.

File Opening Basics:

# Basic file opening
file = open("example.txt", "r")  # Open for reading
content = file.read()
file.close()  # Always close the file!

# Better: Using 'with' statement (auto-closes)
with open("example.txt", "r") as file:
    content = file.read()
# File is automatically closed here

# File modes:
# 'r' - read (default)
# 'w' - write (overwrites existing)
# 'a' - append (adds to end)
# 'x' - exclusive creation (fails if exists)
# 'b' - binary mode
# 't' - text mode (default)
# '+' - read and write

# Common combinations
with open("file.txt", "w") as f:    # Write text
with open("file.txt", "a") as f:    # Append text
with open("file.bin", "wb") as f:   # Write binary
with open("file.txt", "r+") as f:   # Read and write

# Specify encoding for text files
with open("file.txt", "r", encoding="utf-8") as f:
    content = f.read()

Lines 2-5: Manual file opening requires explicit close().

Lines 8-11: The with statement automatically closes the file - always prefer this!

Lines 14-22: File modes control how the file is opened.

Lines 30-31: Always specify encoding for text files to avoid issues.

💡 Always Use 'with'

The with statement is called a context manager. It guarantees the file is closed even if an exception occurs, preventing resource leaks and file corruption.

Reading Files

Python provides multiple ways to read file content depending on your needs.

Reading Methods:

# Read entire file as string
with open("example.txt", "r") as f:
    content = f.read()
    print(content)

# Read specific number of characters
with open("example.txt", "r") as f:
    first_10 = f.read(10)  # Read 10 characters
    rest = f.read()        # Read remaining

# Read one line at a time
with open("example.txt", "r") as f:
    line = f.readline()
    while line:
        print(line, end='')  # line already has \n
        line = f.readline()

# Read all lines into list
with open("example.txt", "r") as f:
    lines = f.readlines()
    # Returns ['line1\n', 'line2\n', ...]

# Iterate over lines (memory efficient)
with open("example.txt", "r") as f:
    for line in f:
        print(line, end='')

# Check if file exists before reading
import os
if os.path.exists("example.txt"):
    with open("example.txt", "r") as f:
        content = f.read()

# Using pathlib (modern approach)
from pathlib import Path
file_path = Path("example.txt")
if file_path.exists():
    content = file_path.read_text()

Lines 2-5: read() loads the entire file into memory.

Lines 8-10: read(n) reads n characters; subsequent reads continue from there.

Lines 13-17: readline() reads one line at a time.

Lines 20-22: readlines() returns a list of all lines.

Lines 25-27: Iterating over the file object is memory-efficient for large files.

Lines 36-38: pathlib provides an object-oriented, cleaner approach.

Writing Files

Write content to files using write mode or append mode.

Writing Methods:

# Write string to file (overwrites)
with open("output.txt", "w") as f:
    f.write("Hello, World!\n")
    f.write("Second line\n")

# Write multiple lines
lines = ["Line 1\n", "Line 2\n", "Line 3\n"]
with open("output.txt", "w") as f:
    f.writelines(lines)  # Note: no automatic newlines

# Append to file
with open("log.txt", "a") as f:
    f.write("New log entry\n")

# Write with print function
with open("output.txt", "w") as f:
    print("Hello", "World", file=f, sep=", ")

# Create directory if needed
from pathlib import Path
Path("subdir/nested").mkdir(parents=True, exist_ok=True)
Path("subdir/nested/file.txt").write_text("Content")

# Write numbers (convert to string first)
numbers = [1, 2, 3, 4, 5]
with open("numbers.txt", "w") as f:
    for num in numbers:
        f.write(f"{num}\n")

# Write and read same file
with open("file.txt", "r+") as f:
    content = f.read()
    f.write("\nAppended content")

Lines 2-5: write() writes strings; doesn't add newlines automatically.

Lines 8-10: writelines() writes a list of strings.

Lines 13-15: Use append mode ('a') to add to existing file.

Lines 18-19: The print() function can write to files.

Lines 22-24: pathlib makes file operations cleaner.

Lines 32-34: r+ mode allows both reading and writing.

Working with CSV Files

CSV (Comma-Separated Values) is a common format for tabular data. Python's csv module makes working with CSV files easy.

CSV Operations:

import csv

# Writing CSV
data = [
    ["Name", "Age", "City"],
    ["Alice", 25, "NYC"],
    ["Bob", 30, "LA"],
    ["Charlie", 35, "Chicago"]
]

with open("people.csv", "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerows(data)

# Reading CSV
with open("people.csv", "r") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)

# CSV with dictionaries
with open("people.csv", "r") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["Name"], row["Age"])

# Write with DictWriter
with open("output.csv", "w", newline="") as f:
    fieldnames = ["name", "age", "city"]
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerow({"name": "Alice", "age": 25, "city": "NYC"})

# Custom delimiter
with open("data.tsv", "w", newline="") as f:
    writer = csv.writer(f, delimiter="\t")
    writer.writerows(data)

Lines 6-13: csv.writer() writes lists as CSV rows.

Lines 16-19: csv.reader() reads CSV rows as lists.

Lines 22-25: DictReader reads rows as dictionaries using header row.

Lines 28-32: DictWriter writes dictionaries with named fields.

Lines 35-37: Use delimiter for different separators (TSV, etc.).

Working with JSON Files

JSON (JavaScript Object Notation) is perfect for structured data and APIs. Python's json module handles serialization seamlessly.

JSON Operations:

import json

# Python dict to JSON string
data = {"name": "Alice", "age": 25, "skills": ["Python", "JavaScript"]}
json_string = json.dumps(data, indent=2)
print(json_string)

# JSON string to Python dict
parsed = json.loads(json_string)
print(parsed["name"])

# Write JSON to file
with open("data.json", "w") as f:
    json.dump(data, f, indent=2)

# Read JSON from file
with open("data.json", "r") as f:
    loaded = json.load(f)

# Handle datetime (not natively JSON serializable)
from datetime import datetime

def json_serializer(obj):
    if isinstance(obj, datetime):
        return obj.isoformat()
    raise TypeError(f"Type {type(obj)} not serializable")

data = {"timestamp": datetime.now()}
json_str = json.dumps(data, default=json_serializer)

# Pretty print with sorting
json.dumps(data, indent=2, sort_keys=True)

# Handle file not found
try:
    with open("config.json", "r") as f:
        config = json.load(f)
except FileNotFoundError:
    config = {"default": True}

Lines 5-6: json.dumps() converts Python objects to JSON string.

Lines 9-10: json.loads() parses JSON string to Python objects.

Lines 13-14: json.dump() writes JSON directly to file.

Lines 17-18: json.load() reads JSON directly from file.

Lines 22-26: Custom serializer for types not natively supported.

Lines 33-37: Handle missing files gracefully.

File Path Operations

The pathlib module provides an object-oriented approach to working with file paths.

Using pathlib:

from pathlib import Path

# Create path objects
current = Path.cwd()  # Current working directory
home = Path.home()    # User's home directory
file_path = Path("folder/subfolder/file.txt")

# Path components
print(file_path.name)      # "file.txt"
print(file_path.stem)      # "file"
print(file_path.suffix)    # ".txt"
print(file_path.parent)    # Path("folder/subfolder")

# Check existence and type
print(file_path.exists())  # False (not created yet)
print(file_path.is_file())
print(file_path.is_dir())

# Create directories
Path("new/dir").mkdir(parents=True, exist_ok=True)

# List directory contents
for item in Path(".").iterdir():
    print(item.name, item.is_dir())

# Glob pattern matching
for py_file in Path(".").glob("**/*.py"):
    print(py_file)

# Read/write files
Path("file.txt").write_text("Hello!")
content = Path("file.txt").read_text()

# Path operations
path1 = Path("folder")
path2 = "subfolder/file.txt"
full = path1 / path2  # Path("folder/subfolder/file.txt")

# Resolve to absolute path
absolute = Path("file.txt").resolve()

Lines 4-6: Create Path objects for different locations.

Lines 9-13: Extract path components easily.

Lines 16-18: Check if path exists and its type.

Lines 25-27: iterdir() iterates over directory contents.

Lines 30-31: glob() finds files matching patterns.

Lines 38-39: The / operator joins paths elegantly.

Binary Files

Binary mode is used for non-text files like images, PDFs, and serialized Python objects.

Binary Operations:

# Read binary file
with open("image.png", "rb") as f:
    data = f.read()
    print(len(data))  # Size in bytes

# Write binary file
with open("copy.png", "wb") as f:
    f.write(data)

# Pickle: Save Python objects
import pickle

data = {"name": "Alice", "scores": [85, 90, 88]}

# Save to file
with open("data.pkl", "wb") as f:
    pickle.dump(data, f)

# Load from file
with open("data.pkl", "rb") as f:
    loaded = pickle.load(f)

# Shelve: Persistent dictionary-like storage
import shelve

with shelve.open("mydb") as db:
    db["key1"] = {"name": "Alice"}
    db["key2"] = [1, 2, 3]
    
with shelve.open("mydb") as db:
    print(db["key1"])

# Read file in chunks (memory efficient for large files)
def read_in_chunks(filename, chunk_size=8192):
    with open(filename, "rb") as f:
        while chunk := f.read(chunk_size):
            yield chunk

for chunk in read_in_chunks("large_file.bin"):
    process(chunk)  # Process each chunk

Lines 2-7: Binary mode ('rb', 'wb') for non-text files.

Lines 10-21: pickle serializes Python objects to disk.

Lines 24-30: shelve provides persistent dictionary storage.

Lines 33-38: Reading in chunks is memory-efficient for large files.

⚠️ Security Warning

Never unpickle data from untrusted sources! Pickle can execute arbitrary code. Use JSON for untrusted data.
