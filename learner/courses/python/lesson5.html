Lesson 5: Loops

Iterate through code with for and while loops, and understand loop control

What are Loops?

Loops allow you to execute a block of code repeatedly. Python has two main types of loops: for loops iterate over sequences, while while loops run as long as a condition is true. Loops are essential for processing collections, repeating actions, and automating tasks.

for Loops

The for loop iterates over a sequence (like a list, string, or range) and executes code for each item. This is Python's most commonly used loop type.

Basic for Loop:

# Iterating over a list
fruits = ["apple", "banana", "cherry"]

for fruit in fruits:
    print(fruit)

# Iterating over a string
for char in "Python":
    print(char)

# Iterating with range()
for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4

# range with start and stop
for i in range(2, 6):
    print(i)  # Prints 2, 3, 4, 5

# range with step
for i in range(0, 10, 2):
    print(i)  # Prints 0, 2, 4, 6, 8

# Countdown using negative step
for i in range(5, 0, -1):
    print(i)  # Prints 5, 4, 3, 2, 1

Lines 3-5: The for loop takes each item from the list and assigns it to the variable fruit.

Lines 8-9: Strings are sequences, so you can iterate over each character.

Lines 12-13: range(n) generates numbers from 0 to n-1.

Lines 16-17: range(start, stop) generates from start to stop-1.

Lines 20-21: The third argument is the step value.

Lines 24-25: Negative step counts downward.

Loop Patterns:

# Using enumerate() for index and value
colors = ["red", "green", "blue"]

for index, color in enumerate(colors):
    print(f"{index}: {color}")

# Starting enumerate at a different number
for index, color in enumerate(colors, start=1):
    print(f"{index}: {color}")

# Looping with index using range
for i in range(len(colors)):
    print(f"colors[{i}] = {colors[i]}")

# Iterating over dictionary
person = {"name": "Alice", "age": 25, "city": "NYC"}

# Keys only (default)
for key in person:
    print(key)

# Keys and values
for key, value in person.items():
    print(f"{key}: {value}")

# Values only
for value in person.values():
    print(value)

# Nested loops
for i in range(3):
    for j in range(3):
        print(f"({i}, {j})", end=" ")
    print()  # New line after each row

Lines 4-5: enumerate() gives both index and value from a sequence.

Lines 8-9: The start parameter sets the starting index.

Lines 18-23: Dictionary iteration options: keys (default), items (key-value pairs), values.

Lines 26-30: Nested loops are useful for multi-dimensional data.

while Loops

while loops execute as long as a condition is true. They're useful when you don't know in advance how many iterations you need.

Basic while Loop:

# Basic while loop
count = 0

while count < 5:
    print(count)
    count += 1  # Don't forget to increment!

# User input validation
password = ""

while len(password) < 8:
    password = input("Enter password (min 8 chars): ")
    if len(password) < 8:
        print("Too short, try again.")

print("Password accepted!")

# Simulating a game loop
playing = True

while playing:
    action = input("Enter action (quit to exit): ")
    
    if action == "quit":
        playing = False
        print("Game over!")
    else:
        print(f"You performed: {action}")

# Finding first number divisible by 7
num = 1

while num % 7 != 0:
    num += 1

print(f"Found: {num}")  # 7

Lines 3-6: while checks the condition before each iteration. Always update the condition variable!

Lines 9-16: while is perfect for input validation loops.

Lines 19-29: Game loops often use a boolean flag with while.

Lines 32-38: Use while when searching for something with unknown iterations.

💡 Infinite Loops

Be careful with while loops! If the condition never becomes false, you get an infinite loop. Always ensure your loop can terminate. Use Ctrl+C to stop an infinite loop.

Loop Control Statements

Python provides break, continue, and else clauses to give you fine control over loop execution.

break, continue, and else:

# break - exit the loop entirely
for i in range(10):
    if i == 5:
        print("Breaking at 5")
        break
    print(i)
# Prints: 0, 1, 2, 3, 4, "Breaking at 5"

# continue - skip to next iteration
for i in range(5):
    if i == 2:
        continue  # Skip 2
    print(i)
# Prints: 0, 1, 3, 4

# Loop else - runs if loop completed without break
for i in range(5):
    if i == 10:
        break
else:
    print("Loop completed normally!")

# Practical example: searching
numbers = [1, 3, 5, 7, 9]
target = 5

for num in numbers:
    if num == target:
        print(f"Found {target}!")
        break
else:
    print(f"{target} not found")

# continue in while loop
i = 0
while i < 10:
    i += 1
    if i % 2 == 0:  # Skip even numbers
        continue
    print(i)  # Prints: 1, 3, 5, 7, 9

Lines 3-7: break immediately exits the entire loop.

Lines 11-14: continue skips the rest of the current iteration and moves to the next.

Lines 17-22: The else block runs only if the loop finished without hitting break.

Lines 25-33: A common pattern: use for-else when searching for items.

Lines 36-41: continue works in while loops too.

List Comprehensions

List comprehensions provide a concise way to create lists from existing sequences. They're a Pythonic alternative to using loops for list creation.

Creating Lists Concisely:

# Traditional loop to create a list
squares = []
for x in range(1, 6):
    squares.append(x ** 2)
print(squares)  # [1, 4, 9, 16, 25]

# List comprehension (same result)
squares = [x ** 2 for x in range(1, 6)]

# With condition (filtering)
evens = [x for x in range(10) if x % 2 == 0]
print(evens)  # [0, 2, 4, 6, 8]

# Transform strings
words = ["hello", "world", "python"]
upper_words = [word.upper() for word in words]
print(upper_words)  # ['HELLO', 'WORLD', 'PYTHON']

# With if-else (different from filter)
labels = ["even" if x % 2 == 0 else "odd" for x in range(5)]
print(labels)  # ['even', 'odd', 'even', 'odd', 'even']

# Nested comprehension (matrix)
matrix = [[i * j for j in range(1, 4)] for i in range(1, 4)]
print(matrix)  # [[1, 2, 3], [2, 4, 6], [3, 6, 9]]

# Flatten a matrix
flat = [num for row in matrix for num in row]
print(flat)  # [1, 2, 3, 2, 4, 6, 3, 6, 9]

# Dictionary comprehension
squares_dict = {x: x**2 for x in range(1, 6)}
print(squares_dict)  # {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

# Set comprehension
unique_lengths = {len(word) for word in words}
print(unique_lengths)  # {5, 6}

Lines 3-6: Traditional approach creates empty list and appends in a loop.

Line 9: List comprehension: [expression for item in iterable].

Line 12: Add condition: [expression for item in iterable if condition].

Line 22: if-else goes before the for: [value_if_true if cond else value_if_false for item in iterable].

Lines 28-29: Nested loops in comprehensions: read left to right.

Lines 32-36: Comprehensions work for dictionaries and sets too.

💡 Readability First

List comprehensions are powerful but can become hard to read. If your comprehension is complex, use a regular loop instead. Readability counts!

The pass Statement

Sometimes you need a placeholder in your loop. The pass statement does nothing but allows syntactically required blocks to be empty.

Using pass:

# pass as placeholder
for i in range(5):
    pass  # TODO: implement later

# While loop with pass
while False:
    pass  # Never runs, but valid syntax

# Empty class or function placeholder
class MyFutureClass:
    pass

def future_function():
    pass

# Useful in try-except when you want to ignore errors
try:
    risky_operation()
except:
    pass  # Silently ignore errors (use sparingly!)

Lines 3-4: pass lets you write syntactically correct but empty loops.

Lines 7-8: Works with all control structures that require a code block.

Lines 11-16: Commonly used as placeholder for classes and functions.

Lines 19-21: Can be used to silently ignore exceptions, but use this sparingly.

Iterators and iter()

Under the hood, for loops work with iterators. Understanding this helps you work with custom iterable objects.

How Iteration Works:

# Manual iteration with iter() and next()
numbers = [1, 2, 3]
iterator = iter(numbers)

print(next(iterator))  # 1
print(next(iterator))  # 2
print(next(iterator))  # 3
# next(iterator) would raise StopIteration

# This is what a for loop does internally
numbers = [1, 2, 3]
iterator = iter(numbers)

while True:
    try:
        num = next(iterator)
        print(num)
    except StopIteration:
        break

# Creating a custom iterator
class Countdown:
    def __init__(self, start):
        self.current = start
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        self.current -= 1
        return self.current + 1

# Using the custom iterator
for num in Countdown(5):
    print(num)  # 5, 4, 3, 2, 1

Lines 3-8: iter() creates an iterator; next() gets the next item.

Lines 11-19: This is essentially what for loops do internally.

Lines 22-36: Custom iterators implement __iter__ and __next__.

Lines 39-41: Once defined, custom iterators work with for loops.
