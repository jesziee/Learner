Lesson 3: Operators

Master arithmetic, comparison, and logical operators in Python

What are Operators?

Operators are special symbols that perform operations on values and variables. Python supports several types of operators including arithmetic, comparison, logical, assignment, and more. Understanding operators is fundamental to writing expressions and making decisions in your code.

Arithmetic Operators

Arithmetic operators perform mathematical operations like addition, subtraction, multiplication, and division.

Basic Arithmetic:

a = 10
b = 3

# Addition
print(a + b)   # 13

# Subtraction
print(a - b)   # 7

# Multiplication
print(a * b)   # 30

# Division (always returns float)
print(a / b)   # 3.3333333333333335

# Floor Division (rounds down to nearest integer)
print(a // b)  # 3

# Modulo (remainder of division)
print(a % b)   # 1

# Exponentiation (power)
print(a ** b)  # 1000 (10 to the power of 3)

# Negative numbers
print(-a)      # -10
print(a + -b)  # 7

# Operator precedence (PEMDAS)
result = 2 + 3 * 4    # 14 (multiplication first)
result = (2 + 3) * 4  # 20 (parentheses first)
result = 2 ** 3 ** 2  # 512 (right to left: 2 ** 9)

Lines 6-17: Basic math operations: +, -, *, / work as expected.

Line 15: Regular division always returns a float, even with integers.

Line 18: Floor division (//) rounds down to the nearest whole number.

Line 21: Modulo (%) returns the remainder - useful for checking even/odd.

Line 24: Double asterisk (**) is the power operator.

Lines 31-33: Python follows mathematical order of operations. Use parentheses to control order.

💡 Tip: Floor Division

Floor division always rounds toward negative infinity, not toward zero. So -7 // 3 equals -3, not -2.

Comparison Operators

Comparison operators compare two values and return a boolean result (True or False). These are essential for making decisions in your programs.

Comparing Values:

x = 5
y = 10

# Equal to
print(x == y)   # False

# Not equal to
print(x != y)   # True

# Greater than
print(x > y)    # False

# Less than
print(x < y)    # True

# Greater than or equal to
print(x >= 5)   # True

# Less than or equal to
print(x <= 4)   # False

# Comparing strings
print("apple" < "banana")  # True (alphabetical order)
print("Apple" < "apple")   # True (uppercase comes before lowercase)
print("abc" == "abc")      # True

# Chained comparisons
age = 25
print(18 <= age <= 65)     # True (age is between 18 and 65)
print(0 < x < 10)          # True (x is between 0 and 10)

Lines 6-21: Standard comparison operators return True or False.

Line 24: Strings compare alphabetically using ASCII/Unicode values.

Line 25: Uppercase letters have lower values than lowercase in ASCII.

Lines 28-30: Python allows chaining comparisons for range checks.

Logical Operators

Logical operators combine boolean values and are used to build complex conditions.

and, or, not:

a = True
b = False

# and - both must be True
print(a and b)    # False
print(a and True) # True

# or - at least one must be True
print(a or b)     # True
print(False or False)  # False

# not - inverts the value
print(not a)      # False
print(not b)      # True

# Practical example
age = 25
has_id = True

# Check if person can enter a club
if age >= 21 and has_id:
    print("Welcome to the club!")

# Check discount eligibility
is_student = False
is_senior = False

if is_student or is_senior:
    print("You get a discount!")
else:
    print("Regular price")

# Short-circuit evaluation
x = 0
# This won't cause an error because x != 0 is False
# Python doesn't evaluate the division when the first part is False
if x != 0 and 10 / x > 1:
    print("This won't print")

Lines 6-7: and returns True only if both operands are True.

Lines 10-11: or returns True if at least one operand is True.

Lines 14-15: not inverts the boolean value.

Lines 18-26: Combine logical operators with comparison operators for complex conditions.

Lines 36-39: Short-circuit evaluation: Python stops evaluating as soon as the result is determined.

Assignment Operators

Assignment operators assign values to variables. Python offers shorthand operators that combine assignment with another operation.

Assignment Shortcuts:

# Basic assignment
x = 10

# Addition assignment
x += 5   # Same as: x = x + 5
print(x)  # 15

# Subtraction assignment
x -= 3   # Same as: x = x - 3
print(x)  # 12

# Multiplication assignment
x *= 2   # Same as: x = x * 2
print(x)  # 24

# Division assignment
x /= 4   # Same as: x = x / 4
print(x)  # 6.0 (becomes float)

# Floor division assignment
x //= 2
print(x)  # 3.0

# Modulo assignment
x %= 2
print(x)  # 1.0

# Exponentiation assignment
x **= 3
print(x)  # 1.0

# Works with strings too
greeting = "Hello"
greeting += " World"
print(greeting)  # Hello World

Lines 2: Basic assignment sets a variable's value.

Lines 5-6: += adds and assigns in one step.

Lines 15-17: Division always produces a float, even with integers.

Lines 29-31: These operators work with strings and other types too.

Identity Operators

Identity operators check if two variables refer to the same object in memory, not just equal values.

is and is not:

# 'is' checks if two variables point to the same object
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)   # True (same values)
print(a is b)   # False (different objects)
print(a is c)   # True (same object)

# Common use: checking for None
value = None
if value is None:
    print("Value is not set")

# Small integers are cached (implementation detail)
x = 256
y = 256
print(x is y)   # True (cached)

x = 257
y = 257
print(x is y)   # May be False (not cached)

# Always use 'is' for None comparisons
# Bad: if value == None
# Good: if value is None

Lines 3-8: is checks object identity, == checks value equality.

Lines 11-13: Use is when checking for None.

Lines 16-23: Python caches small integers (-5 to 256), so they may share identity.

Lines 26-27: Always use is for None, not ==.

Membership Operators

Membership operators test whether a value is contained in a sequence like a string, list, or tuple.

in and not in:

# Check if value is in a string
text = "Hello, Python!"
print("Python" in text)     # True
print("python" in text)     # False (case-sensitive)
print("Java" not in text)   # True

# Check if value is in a list
fruits = ["apple", "banana", "cherry"]
print("apple" in fruits)    # True
print("grape" in fruits)    # False

# Check if key is in a dictionary
person = {"name": "Alice", "age": 25}
print("name" in person)     # True (checks keys)
print("Alice" in person)    # False (not a key)
print("age" in person)      # True

# Practical use in conditionals
user_input = "yes"
if user_input in ["yes", "y", "Yeah"]:
    print("User agreed!")

# Check if character is a vowel
char = "e"
if char in "aeiou":
    print(f"{char} is a vowel")

Lines 3-6: in checks for substrings in strings.

Lines 9-11: Check for items in a list.

Lines 14-17: For dictionaries, in checks keys, not values.

Lines 20-22: Useful for checking against multiple valid values.

Bitwise Operators

Bitwise operators work on individual bits of integers. They're useful for low-level programming and optimization.

Working with Bits:

a = 10  # Binary: 1010
b = 4   # Binary: 0100

# AND (&) - 1 only if both bits are 1
print(a & b)   # 0 (0000)

# OR (|) - 1 if either bit is 1
print(a | b)   # 14 (1110)

# XOR (^) - 1 if bits are different
print(a ^ b)   # 14 (1110)

# NOT (~) - inverts all bits
print(~a)      # -11 (two's complement)

# Left shift (<<) - shifts bits left, adds zeros
print(a << 2)  # 40 (101000)

# Right shift (>>) - shifts bits right
print(a >> 1)  # 5 (0101)

# Practical use: checking flags
READ = 1      # 001
WRITE = 2     # 010
EXECUTE = 4   # 100

permissions = READ | WRITE  # 011

if permissions & READ:
    print("Has read permission")
if permissions & EXECUTE:
    print("Has execute permission")  # Won't print

Lines 4-14: Basic bitwise operations: AND, OR, XOR, NOT.

Lines 17-19: Shift operators move bits left or right.

Lines 22-33: Bitwise operators are commonly used for flags and permissions.
