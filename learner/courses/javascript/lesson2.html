Lesson 2: Variables

Learn how to store and manage data using variables in JavaScript.

What are Variables?

Variables are containers for storing data values. Think of them as labeled boxes where you can put information that you want to use later. In JavaScript, we use variables to store all kinds of data: numbers, text, lists, and more.

Analogy: Think of a variable like a storage box. You put a label on the box (the variable name) and put something inside (the value). Later, you can find the box by its label and retrieve what's inside.

Declaring Variables

In JavaScript, there are three ways to declare variables:

1. Using let

The modern way to declare variables that can be changed:

let age = 25;
console.log(age);  // Outputs: 25

age = 26;          // We can change the value
console.log(age);  // Outputs: 26
Code Explanation:
  • let age = 25; - Declares a variable named 'age' with value 25
  • age = 26; - Changes the value to 26 (reassignment)
  • let allows you to change the value later

2. Using const

For variables that should never change:

const PI = 3.14159;
const MAX_SIZE = 100;

console.log(PI);       // Outputs: 3.14159

// PI = 3.14;          // ERROR! Cannot change a const
Code Explanation:
  • const PI = 3.14159; - Creates a constant (unchangeable) variable
  • Constants must be assigned a value when declared
  • Attempting to change a const causes an error
  • Use const by default, let only when you need to reassign

3. Using var (Old Way)

The original way to declare variables (mostly avoid in modern JavaScript):

var name = "John";
var name = "Jane";    // Can redeclare (not ideal)

console.log(name);    // Outputs: Jane
Best Practice: Avoid var in modern JavaScript. Use const for values that won't change and let for values that will change.

Naming Variables

Variable names must follow certain rules:

// ✅ Valid variable names
let userName = "Alice";      // Letters and numbers
let _private = "hidden";     // Underscore allowed
let $price = 99.99;          // Dollar sign allowed
let camelCase = "popular";   // Camel case (recommended)
let userAge = 25;

// ❌ Invalid variable names
// let 2fast = "too fast";   // Cannot start with number
// let my-var = "invalid";   // Hyphens not allowed
// let let = "reserved";     // Cannot use reserved words
Naming Rules:
  • Names can contain letters, digits, underscores, and dollar signs
  • Names must begin with a letter, underscore, or dollar sign
  • Names are case sensitive (age and Age are different)
  • Reserved words (like let, const, if) cannot be used as names

Naming Conventions

// Camel Case (recommended in JavaScript)
let firstName = "John";
let lastName = "Doe";
let totalPrice = 99.99;

// Descriptive names are better
let x = 60;                        // Not clear what x means
let secondsInMinute = 60;          // Much clearer!

// Boolean variables often start with is, has, or should
let isLoggedIn = true;
let hasPermission = false;
let shouldDisplay = true;

Data Types

Variables can hold different types of data:

Numbers

let integer = 42;           // Whole number
let decimal = 3.14;         // Decimal number
let negative = -10;         // Negative number
let scientific = 2.5e6;     // Scientific notation (2,500,000)

// Numbers can be used in calculations
let sum = integer + decimal;  // 45.14

Strings (Text)

// Strings can use single or double quotes
let singleQuotes = 'Hello';
let doubleQuotes = "World";

// Template literals (modern way)
let greeting = `Hello, ${singleQuotes}!`;
console.log(greeting);  // Outputs: Hello, Hello!

// Strings can be combined (concatenation)
let fullName = 'John' + ' ' + 'Doe';
console.log(fullName);  // Outputs: John Doe
Code Explanation:
  • `Hello, ${name}!` - Template literal with variable embedding
  • ${variable} - Placeholder that gets replaced with the variable's value
  • Backticks (`) are required for template literals

Booleans

let isJavaScriptFun = true;
let isBoring = false;

// Booleans are used for decisions
if (isJavaScriptFun) {
    console.log("Let's learn more!");
}

Undefined and Null

let notAssigned;            // undefined (no value assigned)
let emptyValue = null;      // null (intentionally empty)

console.log(notAssigned);   // Outputs: undefined
console.log(emptyValue);    // Outputs: null
Code Explanation:
  • undefined - Variable declared but no value assigned
  • null - Intentionally empty value (represents "nothing")
  • Both represent "no value" but are used differently

Variable Scope

Where you declare a variable determines where it can be used:

// Global scope - accessible everywhere
let globalVar = "I'm global";

function myFunction() {
    // Function scope - only inside this function
    let localVar = "I'm local";
    
    console.log(globalVar);  // ✅ Works
    console.log(localVar);   // ✅ Works
}

myFunction();

console.log(globalVar);      // ✅ Works
// console.log(localVar);    // ❌ Error! Not accessible here
Scope Types:
  • Global scope: Variables declared outside any function/block
  • Function scope: Variables declared inside a function
  • Block scope: Variables declared inside { } with let/const

Multiple Declarations

You can declare multiple variables at once:

// Multiple variables on one line
let a = 1, b = 2, c = 3;

// Or on multiple lines (more readable)
let firstName = "John",
    lastName = "Doe",
    age = 30,
    city = "New York";

// Or declare first, assign later
let x, y, z;
x = 1;
y = 2;
z = 3;

Practice Exercise

Create variables to store information about yourself:

// Try this exercise
const fullName = "Your Name";
let age = 25;
let isStudent = true;
let favoriteLanguage = "JavaScript";

console.log("Name:", fullName);
console.log("Age:", age);
console.log("Is Student:", isStudent);
console.log("Favorite Language:", favoriteLanguage);

// Now try using template literals
