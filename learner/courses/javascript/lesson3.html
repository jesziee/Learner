Lesson 3: Operators

Learn how to perform calculations and comparisons in JavaScript.

What are Operators?

Operators are symbols that tell JavaScript to perform actions on values. Just like in math, you use operators to add, subtract, compare, and manipulate data. JavaScript has several types of operators that we'll explore in this lesson.

Arithmetic Operators

These operators perform mathematical calculations:

let a = 10;
let b = 3;

// Basic arithmetic
console.log(a + b);   // Addition: 13
console.log(a - b);   // Subtraction: 7
console.log(a * b);   // Multiplication: 30
console.log(a / b);   // Division: 3.333...
console.log(a % b);   // Modulus (remainder): 1
console.log(a ** b);  // Exponent (power): 1000
Operator Breakdown:
  • + Addition - adds two values together
  • - Subtraction - subtracts right from left
  • * Multiplication - multiplies two values
  • / Division - divides left by right
  • % Modulus - returns the remainder after division
  • ** Exponent - raises to the power of

The Modulus Operator

The modulus operator (%) is particularly useful for checking if a number is even or odd:

let number = 7;

// Check if number is even or odd
if (number % 2 === 0) {
    console.log("Even number");
} else {
    console.log("Odd number");  // This will print
}

// Useful for cycling through values
let index = 0;
index = (index + 1) % 3;  // Cycles: 0, 1, 2, 0, 1, 2...

Increment and Decrement

let count = 5;

count++;     // Increment: count is now 6
console.log(count);  // 6

count--;     // Decrement: count is now 5
console.log(count);  // 5

// Prefix vs Postfix
let x = 5;
console.log(x++);  // Outputs 5, then x becomes 6
console.log(x);    // Outputs 6

let y = 5;
console.log(++y);  // y becomes 6, then outputs 6
Prefix vs Postfix:
  • x++ (Postfix) - Returns current value, then increments
  • ++x (Prefix) - Increments first, then returns new value

Assignment Operators

These operators assign values to variables:

let x = 10;      // Basic assignment

x += 5;          // Same as: x = x + 5   (x is now 15)
x -= 3;          // Same as: x = x - 3   (x is now 12)
x *= 2;          // Same as: x = x * 2   (x is now 24)
x /= 4;          // Same as: x = x / 4   (x is now 6)
x %= 4;          // Same as: x = x % 4   (x is now 2)
x **= 3;         // Same as: x = x ** 3  (x is now 8)
Compound Assignment:
  • These are shortcuts that combine an operation with assignment
  • x += 5 is shorter than x = x + 5
  • Makes code more concise and readable

Comparison Operators

These operators compare values and return true or false:

let a = 5;
let b = "5";
let c = 10;

// Equality
console.log(a == b);   // true (loose equality - checks value)
console.log(a === b);  // false (strict equality - checks value AND type)
console.log(a != b);   // false (loose not equal)
console.log(a !== b);  // true (strict not equal - different types)

// Comparison
console.log(a > 3);    // true (greater than)
console.log(a < 10);   // true (less than)
console.log(a >= 5);   // true (greater than or equal)
console.log(a <= 4);   // false (less than or equal)
Equality Operators:
  • == Loose equality - converts types then compares
  • === Strict equality - compares both value AND type
  • != Loose not equal
  • !== Strict not equal
Best Practice: Always use === and !== for comparisons. They prevent unexpected results caused by type conversion.

Type Conversion Issues

// Why strict equality matters
console.log(5 == "5");    // true (string converted to number)
console.log(5 === "5");   // false (different types)

console.log(0 == false);   // true (false converts to 0)
console.log(0 === false);  // false (different types)

console.log(null == undefined);   // true (special case)
console.log(null === undefined);  // false (different types)

// The safe approach
if (a === 5) {
    console.log("a is exactly the number 5");
}

Logical Operators

These operators work with boolean values (true/false):

let isLoggedIn = true;
let hasPermission = false;

// AND (&&) - both must be true
console.log(isLoggedIn && hasPermission);  // false

// OR (||) - at least one must be true
console.log(isLoggedIn || hasPermission);  // true

// NOT (!) - reverses the boolean
console.log(!isLoggedIn);    // false
console.log(!hasPermission); // true
Logical Operator Truth Tables:
  • AND (&&): true && true = true, everything else = false
  • OR (||): false || false = false, everything else = true
  • NOT (!): !true = false, !false = true

Practical Examples

let age = 25;
let hasID = true;

// Can enter the club?
let canEnter = age >= 21 && hasID;
console.log(canEnter);  // true

// Is it a weekend?
let day = "Saturday";
let isWeekend = day === "Saturday" || day === "Sunday";
console.log(isWeekend);  // true

// Is user NOT an admin?
let role = "user";
let isNotAdmin = !(role === "admin");
console.log(isNotAdmin);  // true

String Operators

The + operator can also be used with strings:

// String concatenation
let firstName = "John";
let lastName = "Doe";

let fullName = firstName + " " + lastName;
console.log(fullName);  // "John Doe"

// Concatenating strings and numbers
let age = 25;
let message = "I am " + age + " years old";
console.log(message);  // "I am 25 years old"

// Note: JavaScript converts numbers to strings when combining
console.log("5" + 3);   // "53" (string concatenation)
console.log(5 + 3);     // 8 (number addition)
Watch Out: When using + with strings and numbers, JavaScript converts numbers to strings. For math with strings, convert them first: Number("5") + 3 equals 8.

The Ternary Operator

A shortcut for simple if-else statements:

// Syntax: condition ? valueIfTrue : valueIfFalse

let age = 20;

// Instead of this:
let status1;
if (age >= 18) {
    status1 = "adult";
} else {
    status1 = "minor";
}

// You can write this:
let status2 = age >= 18 ? "adult" : "minor";

console.log(status2);  // "adult"

// Another example
let points = 75;
let result = points >= 50 ? "Pass" : "Fail";
console.log(result);  // "Pass"
Ternary Syntax:
  • condition ? - The condition to check
  • valueIfTrue : - Value returned if condition is true
  • valueIfFalse - Value returned if condition is false

Operator Precedence

Operators have an order of operations, just like in math:

// Multiplication happens before addition
let result = 2 + 3 * 4;
console.log(result);  // 14 (not 20)

// Use parentheses to control order
let result2 = (2 + 3) * 4;
console.log(result2);  // 20

// Complex example
let x = 10;
let y = 5;
let z = 2;

let answer = x + y * z - x / y;
// Step by step:
// 1. y * z = 10
// 2. x / y = 2
// 3. x + 10 - 2 = 18
console.log(answer);  // 18
Precedence Order (highest to lowest):
  • Parentheses ()
  • Exponentiation **
  • Multiplication/Division * / %
  • Addition/Subtraction + -
  • Comparison == === > <
  • Logical AND &&
  • Logical OR ||
