Lesson 4: Conditionals

Learn how to make decisions in your code with if statements and switches.

What are Conditionals?

Conditionals allow your code to make decisions. Just like in real life, programs often need to take different actions based on different conditions. "If it's raining, take an umbrella. Otherwise, leave it at home." JavaScript conditionals work the same way!

Key Concept: Conditionals use boolean values (true/false) to decide which code to run. The condition is evaluated, and if it's true, certain code executes.

The if Statement

The most basic conditional - runs code only if a condition is true:

// Basic if statement
let temperature = 30;

if (temperature > 25) {
    console.log("It's hot outside!");
}

// The condition (temperature > 25) is true, so the message prints
Code Explanation:
  • if - Keyword that starts the conditional
  • (temperature > 25) - The condition to check (must be true/false)
  • { } - Curly braces contain the code to run if condition is true
  • Code inside { } only runs when temperature is greater than 25

Real-World Example

let userAge = 18;

if (userAge >= 18) {
    console.log("You can vote!");
}

if (userAge >= 21) {
    console.log("You can drink alcohol in the US!");
}

// Output: "You can vote!" (but not the second message)

The else Statement

What if you want to run different code when the condition is false? Use else:

let isRaining = false;

if (isRaining) {
    console.log("Take an umbrella!");
} else {
    console.log("Enjoy the sunny day!");
}

// Output: "Enjoy the sunny day!"
Code Explanation:
  • if (isRaining) - Checks if isRaining is true
  • Since isRaining is false, the first block is skipped
  • else - Runs when the if condition is false
  • The else block always runs when the if condition is false

Password Validation Example

let password = "secret123";
let confirmPassword = "secret456";

if (password === confirmPassword) {
    console.log("Password matches! Account created.");
} else {
    console.log("Passwords don't match. Please try again.");
}

// Output: "Passwords don't match. Please try again."

The else if Statement

Check multiple conditions in sequence:

let score = 75;

if (score >= 90) {
    console.log("Grade: A - Excellent!");
} else if (score >= 80) {
    console.log("Grade: B - Good job!");
} else if (score >= 70) {
    console.log("Grade: C - Satisfactory");
} else if (score >= 60) {
    console.log("Grade: D - Needs improvement");
} else {
    console.log("Grade: F - Please study more");
}

// Output: "Grade: C - Satisfactory"
How it works:
  • First, check if score >= 90 (false)
  • Then check if score >= 80 (false)
  • Then check if score >= 70 (true!) - runs this block
  • Remaining conditions are skipped
  • Only ONE block ever executes

Time of Day Example

let hour = 14;  // 24-hour format

if (hour < 12) {
    console.log("Good morning!");
} else if (hour < 17) {
    console.log("Good afternoon!");
} else if (hour < 21) {
    console.log("Good evening!");
} else {
    console.log("Good night!");
}

// Output: "Good afternoon!" (14 is between 12 and 17)

Nested Conditionals

You can place conditionals inside other conditionals:

let age = 25;
let hasLicense = true;

if (age >= 18) {
    console.log("You're an adult");
    
    if (hasLicense) {
        console.log("You can drive!");
    } else {
        console.log("You need a license to drive");
    }
} else {
    console.log("You're a minor");
}

// Output:
// "You're an adult"
// "You can drive!"
Code Explanation:
  • First, check if age >= 18 (true)
  • Inside that block, check if hasLicense (true)
  • Both conditions must be true for "You can drive!"
  • Nested conditionals are checked only if the outer condition is true
Best Practice: Avoid deeply nested conditionals (more than 2-3 levels). They make code hard to read. Instead, use logical operators or return early.

Logical Operators with Conditionals

Combine multiple conditions using AND (&&) and OR (||):

Using AND (&&)

let age = 25;
let hasID = true;

// Both conditions must be true
if (age >= 21 && hasID) {
    console.log("Welcome to the club!");
} else {
    console.log("Sorry, you can't enter");
}

// Output: "Welcome to the club!"

// Example: User login
let username = "admin";
let password = "secret";
let isActive = true;

if (username === "admin" && password === "secret" && isActive) {
    console.log("Login successful!");
}

Using OR (||)

let day = "Saturday";

// At least one condition must be true
if (day === "Saturday" || day === "Sunday") {
    console.log("It's the weekend!");
} else {
    console.log("It's a weekday");
}

// Output: "It's the weekend!"

// Example: Admin access
let role = "editor";

if (role === "admin" || role === "editor") {
    console.log("You have editing permissions");
}

Combining AND and OR

let isMember = true;
let purchaseAmount = 150;
let hasCoupon = false;

// Members get free shipping OR orders over $100
if (isMember || purchaseAmount > 100) {
    console.log("Free shipping applied!");
}

// More complex: VIP members OR (non-VIP with coupon)
let isVIP = false;

if (isVIP || (!isVIP && hasCoupon)) {
    console.log("Discount applied!");
} else {
    console.log("No discount available");
}

The switch Statement

A cleaner way to check one value against many possibilities:

let dayNumber = 3;
let dayName;

switch (dayNumber) {
    case 1:
        dayName = "Monday";
        break;
    case 2:
        dayName = "Tuesday";
        break;
    case 3:
        dayName = "Wednesday";
        break;
    case 4:
        dayName = "Thursday";
        break;
    case 5:
        dayName = "Friday";
        break;
    case 6:
        dayName = "Saturday";
        break;
    case 7:
        dayName = "Sunday";
        break;
    default:
        dayName = "Invalid day";
}

console.log(dayName);  // "Wednesday"
Switch Syntax:
  • switch (value) - The value to compare against
  • case 1: - If value equals 1, run this code
  • break; - Exit the switch (important!)
  • default: - Runs if no case matches (like else)

The Break Statement

Without break, code "falls through" to the next case:

let grade = "B";

switch (grade) {
    case "A":
        console.log("Excellent!");
        break;
    case "B":
        console.log("Good job!");
        // Missing break - falls through!
    case "C":
        console.log("Satisfactory");
        break;
}

// Output:
// "Good job!"
// "Satisfactory"  (unintended!)

Grouping Cases

let month = 2;

switch (month) {
    case 1:
    case 3:
    case 5:
    case 7:
    case 8:
    case 10:
    case 12:
        console.log("31 days");
        break;
    case 4:
    case 6:
    case 9:
    case 11:
        console.log("30 days");
        break;
    case 2:
        console.log("28 or 29 days");
        break;
}

// Output: "28 or 29 days"

Truthy and Falsy Values

In JavaScript, values can be "truthy" or "falsy" in conditionals:

// Falsy values (treated as false)
if (false) { }      // false
if (0) { }          // zero is falsy
if ("") { }         // empty string is falsy
if (null) { }       // null is falsy
if (undefined) { }  // undefined is falsy
if (NaN) { }        // Not a Number is falsy

// Truthy values (treated as true)
if (true) { }       // true
if (1) { }          // any non-zero number
if ("hello") { }    // non-empty string
if ([]) { }         // arrays (even empty)
if ({}) { }         // objects (even empty)

// Practical use
let userName = "";

if (userName) {
    console.log("Hello, " + userName);
} else {
    console.log("Please enter your name");
}
// Output: "Please enter your name"
Why This Matters:
  • You can check if a variable has a value without explicit comparison
  • if (name) is shorter than if (name !== "" && name !== null)
  • Be careful with 0 and empty strings - they're valid values but falsy!

Short-Circuit Evaluation

Logical operators can be used for conditional logic:

// Using && for conditional execution
let isLoggedIn = true;

isLoggedIn && console.log("Welcome back!");
// Runs only if isLoggedIn is true

// Using || for default values
let userName = "";
let displayName = userName || "Guest";
console.log(displayName);  // "Guest" (userName is falsy)

userName = "Alice";
displayName = userName || "Guest";
console.log(displayName);  // "Alice"

// Modern nullish coalescing (??)
// Only uses default for null/undefined, not other falsy values
let count = 0;
let displayCount = count ?? "No items";
console.log(displayCount);  // 0 (not "No items")

Practice Exercise

Build a simple calculator that determines grade and pass/fail:

// Exercise: Grade Calculator
let score = 85;
let attendance = 90;

// Determine grade
let grade;
if (score >= 90) {
    grade = "A";
} else if (score >= 80) {
    grade = "B";
} else if (score >= 70) {
    grade = "C";
} else if (score >= 60) {
    grade = "D";
} else {
    grade = "F";
}

// Check if passed (grade D or better AND attendance >= 75%)
let passed = (grade !== "F") && (attendance >= 75);

console.log("Grade:", grade);
console.log("Passed:", passed);

    case "Sunday":

console.log(day + " is a " + dayType);
