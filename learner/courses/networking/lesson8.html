🛠️ Lesson 8: Troubleshooting

Systematic network diagnosis: ping, traceroute, Wireshark, common fixes

Troubleshooting Methodology

  1. Define symptoms
  2. Reproduce issue
  3. Bottom-up OSI testing
  4. Fix and verify

80% issues: Cable/DNS/WiFi password.

Ping: Layer 3+ Test

ICMP echo request/reply. Tests reachability.

Ping Examples:

ping 8.8.8.8          # IP reachability
ping google.com       # DNS + IP
ping -c 4 127.0.0.1   # Loopback (Linux)
ping -n 4 127.0.0.1   # Loopback (Windows)

Results: time=10ms, packet loss

Traceroute: Path Discovery

Shows route hops + latency.

traceroute google.com (Linux/Mac)
tracert google.com (Windows)

1  192.168.1.1 (1ms)
2  10.0.0.1 (8ms) 
3  * * * (timeout)
...

Wireshark: Packet Capture

Protocol analyzer. Capture/filter packets live.

Filter: ip.addr == 8.8.8.8 Follow TCP stream

Common Issues & Fixes

Issue Checklist:

No Internet:
• Check cables/WiFi
• IP config: ipconfig / release & renew
• DNS: 8.8.8.8 / 1.1.1.1

Slow Network:
• Channel interference
• Duplex mismatch
• MTU issues

Intermittent:
• ARP cache: arp -d *
• Firewall blocks

Practice: Connectivity Tester

Network Diagnostic Tool:

async function diagnoseNetwork() {
  console.log('🩺 Network Diagnosis Starting...');
  
  // Test 1: Local
  try {
    await fetch('data:text/plain;base64,local');
    console.log('✅ Local: OK');
  } catch {
    console.log('❌ Local: FAILED');
    return;
  }
  
  // Test 2: DNS
  try {
    await fetch('https://httpbin.org/ip');
    console.log('✅ DNS + Internet: OK');
  } catch(e) {
    console.log('❌ DNS/Internet: ' + e.message);
  }
  
  // Test 3: Latency
  const start = performance.now();
  await fetch('https://httpbin.org/delay/1');
  console.log(`⏱️ Latency: ${(performance.now()-start).toFixed(0)}ms`);
}
diagnoseNetwork();

Key Takeaways

  • Ping tests reachability (ICMP)
  • Traceroute shows path hops
  • Wireshark captures packets
  • Check physical → IP → DNS order
  • ipconfig renew, change DNS common fixes
