🔗 Lesson 5: TCP & UDP Protocols

Reliable delivery (TCP) vs speed (UDP), ports, handshakes, sockets explained

Layer 4: Transport Layer

End-to-end delivery between applications. Adds ports + reliability.

TCP: Reliable, ordered UDP: Fast, fire-and-forget

TCP Three-Way Handshake

Connection Setup:

1. Client → SYN → Server (seq=x)
2. Server → SYN-ACK → Client (seq=y, ack=x+1)  
3. Client → ACK → Server (ack=y+1)

✅ Connection established

Sequence numbers prevent old packets.

TCP Reliability Features

  • ACKs confirm receipt
  • Retransmit lost packets
  • Flow control (window size)
  • Congestion control

UDP: Connectionless

No handshake, no ACKs, no order. Minimal overhead.

UDP Use Cases:
• Video streaming (Netflix)
• Online gaming (low latency)
• DNS queries
• VoIP calls

Ports & Sockets

Socket = IP:Port (192.168.1.10:80)

Common Ports:

80: HTTP
443: HTTPS
53: DNS
123: NTP
1935: RTMP (video)

Practice: Port Scanner Simulation

UDP/TCP Service Detection:

async function testServices() {
  const services = [
    {port: 80, name: 'HTTP'},
    {port: 443, name: 'HTTPS'},
    {port: 53, name: 'DNS (UDP)'}
  ];
  
  for (const svc of services) {
    try {
      const controller = new AbortController();
      setTimeout(() => controller.abort(), 2000);
      
      const url = `https://httpbin.org/delay/1`;
      await fetch(url, {signal: controller.signal});
      console.log(`✅ ${svc.name} reachable (port ${svc.port})`);
    } catch(e) {
      console.log(`❌ ${svc.name} (port ${svc.port}): ${e.message}`);
    }
  }
}
testServices();

Key Takeaways

  • TCP: SYN→SYN-ACK→ACK, reliable ordered delivery
  • UDP: Fast, unreliable, connectionless
  • Socket = IP:PORT
  • HTTP=80, HTTPS=443, DNS=53
  • TCP for files/web, UDP for video/gaming
