☁️ Lesson 11: Cloud Networking

VPCs, SDN, AWS/GCP/Azure networking, hybrid cloud, Direct Connect

Virtual Private Cloud (VPC)

Logically isolated network section in cloud. Your private data center.

Subnets, route tables, gateways Public/private subnets

Software Defined Networking (SDN)

Control plane separate from data plane. Programmable networks.

OpenFlow separates routing logic from switches.

AWS Networking

AWS Components:

VPC: Virtual network
IGW:  Internet Gateway
NAT:  Private subnet Internet
ELB:  Elastic Load Balancer
Route53: AWS DNS
Direct Connect: Private fiber

GCP Networking

VPC: Global scope
Cloud Router: Dynamic routing
Cloud Load Balancer: L4-L7
Cloud VPN: IPsec tunnels
Interconnect: Private links

Azure Networking

VNet: Virtual Network
NSG: Network Security Groups
Application Gateway: L7 LB
ExpressRoute: Private to Azure
Virtual WAN: Global connectivity

Hybrid Cloud Patterns

Site-to-cloud connectivity.

  • VPN: Encrypted tunnels (cost-effective)
  • Dedicated: Direct Connect/MPLS (low latency)
  • SD-WAN: Intelligent path selection

Practice: Cloud API Latency

Region Ping Test:

async function testCloudLatency() {
  const clouds = {
    aws: 'https://s3.amazonaws.com',
    gcp: 'https://storage.googleapis.com',
    azure: 'https://storage.blob.core.windows.net'
  };
  
  console.log('☁️ Cloud Provider Latency Test');
  
  for (const [provider, url] of Object.entries(clouds)) {
    try {
      const start = performance.now();
      await fetch(url, {method: 'HEAD', mode: 'no-cors'});
      const latency = performance.now() - start;
      console.log(`${provider.padEnd(6)}: ${latency.toFixed(0)}ms`);
    } catch(e) {
      console.log(`${provider.padEnd(6)}: Error`);
    }
  }
}
testCloudLatency();

Key Takeaways

  • VPC = isolated cloud network
  • SDN: Programmable control plane
  • AWS: VPC/IGW/ELB/Direct Connect
  • GCP: Global VPC/Cloud Router
  • Hybrid: VPN + Dedicated links
