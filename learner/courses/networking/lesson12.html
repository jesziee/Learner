📊 Lesson 12: Monitoring & Automation

Network observability, SNMP, Prometheus, Ansible, NetFlow - enterprise operations

Observability Pillars

Metrics + Logs + Traces = Full picture.

Red: Errors/alerts Yellow: Warnings/slow Green: Healthy

SNMP (Simple Network Management)

Standard protocol for device polling. OID tree.

SNMP Components:

Manager: Nagios/Zabbix
Agent:  Device software  
OID:    .1.3.6.1.2.1.1.1.0 (sysDescr)

v1: Community string
v3: Authentication/encryption

Polling + traps (alerts).

NetFlow/sFlow/IPFIX

Traffic flow analysis (who, what, where, how much).

  • NetFlow v9: Cisco
  • sFlow: Sampling
  • Exporters → Collectors → Visualizers

Modern Monitoring: Prometheus

Pull model, time-series, alerting rules.

metric: node_network_receive_bytes_total
query: rate(http_requests_total[5m])
alert: latency > 500ms

Network Automation: Ansible

Configuration management. Playbooks for routers/switches.

---
- name: Configure router
  hosts: routers  
  tasks:
    - ios_config:
        lines: 
          - ip route 0.0.0.0 0.0.0.0 8.8.8.8

SIEM & Observability Platforms

ELK Stack, Splunk, Grafana, Datadog.

Practice: Performance Monitor

Real-time Network Metrics:

let startTime = performance.now();
let requests = 0;

async function monitorNetwork() {
  console.log('📈 Network Monitor Started');
  
  setInterval(async () => {
    const start = performance.now();
    try {
      await fetch('https://httpbin.org/delay/1');
      requests++;
      const rtt = performance.now() - start;
      console.log(`Ping: ${rtt.toFixed(0)}ms | Total: ${requests}`);
    } catch(e) {
      console.log(`Error: ${e.message}`);
    }
  }, 1000);
}

monitorNetwork();

Key Takeaways

  • SNMP: Device management (v3 secure)
  • NetFlow: Traffic analytics
  • Prometheus: Metrics/alerting
  • Ansible: Infrastructure as code
  • Observability > Monitoring
