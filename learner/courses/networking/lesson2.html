📍 Lesson 2: IP Addressing & Subnets

Master IPv4/IPv6 addressing, subnetting, CIDR - calculate networks like a pro

Why IP Addresses?

Unique identifiers for devices on networks, like postal addresses for data packets.

Analogy: IP = home address, MAC = apartment number.

IPv4 Format

32-bit address in dotted decimal: 192.168.1.1

Binary vs Decimal:

Decimal: 192.168.1.1
Binary:  11000000.10101000.00000001.00000001

Each octet = 8 bits (0-255). Total 2^32 ≈ 4.3B addresses.

Public vs Private IPs

Public: Internet routable (8.8.8.8) Private: LAN only (192.168.x.x) 
Private Ranges:
10.0.0.0 - 10.255.255.255
172.16.0.0 - 172.31.255.255
192.168.0.0 - 192.168.255.255

Subnet Masks & CIDR

Mask defines network vs host portion. CIDR: /24 = 255.255.255.0

Subnet Examples:

/24 (255.255.255.0)  → 256 addresses (254 usable)
 /24 → 65,536 addresses
 /16 → 65,536 addresses

IPv6: The Future

128-bit hexadecimal: 2001:db8::1

💡 Solves IPv4 exhaustion (2^128 addresses).

Practice: IP Tools

  try {
    const res = await fetch('https://httpbin.org/ip');
    const data = await res.json();
    console.log('Your Public IP:', data.origin);
    
    // Simple subnet calc
    const ip = '192.168.1.100';
    const cidr = 24;
    const hosts = 2 ** (32 - cidr) - 2;
    console.log(`/${cidr} has ${hosts} usable hosts`);
  } catch(e) {
    console.log('Network error');
  }
}
getMyIP();

Key Takeaways

  • IPv4: 192.168.1.1 (dotted decimal)
  • Private ranges: 192.168.x.x, 10.x.x.x
  • CIDR /24 = 254 hosts
  • IPv6: Hex, massive address space
  • Network = IP & Mask
