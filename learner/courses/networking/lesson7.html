🔍 Lesson 7: DNS & Name Resolution

Domain Name System hierarchy, records, caching, troubleshooting - internet's phonebook

What is DNS?

Hierarchical distributed database translating domains to IPs (google.com → 142.250.190.14).

UDP port 53. 350B queries daily!

DNS Hierarchy

Root → TLD (.com) → Authoritative → Recursive resolver.

Resolution Chain:

1. Root (.): Knows TLD servers
2. TLD (.com): Knows domain servers  
3. Authoritative: Has the IP record
4. Recursive: Your ISP/local resolver

DNS Record Types

Common Records:

A:     IPv4 (google.com → IP)
AAAA:  IPv6
CNAME: Alias (www → google.com)
MX:    Mail server
NS:    Name servers
TXT:   Verification/SPF

TTL controls caching duration.

DNS Caching

Resolvers/OS cache responses (performance). Can cause stale data.

Browser → OS → Resolver → Authoritative

DNS Security (DNSSEC)

DNSSEC adds cryptographic signatures. DoH/DoT encrypts queries.

Practice: DNS Lookup

DNS Query Simulation:

async function dnsLookup(domain) {
  try {
    // Simulate recursive resolution
    const ips = {
      'google.com': '142.250.190.14',
      'github.com': '140.82.121.3',
      'cloudflare.com': '104.16.248.249'
    };
    
    const ip = ips[domain] || 'Unknown';
    console.log(`✅ ${domain} resolves to ${ip}`);
    
    // Real fetch test
    await fetch(`https://${domain}/robots.txt`, {method: 'HEAD'});
    console.log(`Site reachable`);
  } catch(e) {
    console.log(`❌ DNS/Connect error: ${e.message}`);
  }
}

dnsLookup('google.com');
dnsLookup('example.com');

Key Takeaways

  • DNS: Domain → IP phonebook (UDP/53)
  • Root → TLD → Authoritative hierarchy
  • A=IPv4, AAAA=IPv6, MX=mail, CNAME=alias
  • Recursive resolver (8.8.8.8) does lookup
  • Caching speeds resolution
