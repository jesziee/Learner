🗺️ Lesson 6: Routing Protocols

Static vs dynamic routing, OSPF, BGP, path selection - how data finds its way

What is Routing?

Layer 3 (IP) process of forwarding packets between networks. Routers make decisions.

Analogy: GPS for packets across internet highways.

Static Routing

Manually configured routes. Simple, predictable.

Static Route Example:

ip route 10.0.0.0 255.255.255.0 192.168.1.1
# Network → Mask → Next Hop

Good for small networks.

Dynamic Routing Protocols

Routers automatically learn routes. Adapt to failures.

IGP vs EGP:

Interior (same org):
• RIP: Distance vector, hop count
• OSPF: Link-state, Dijkstra SPF
• EIGRP: Cisco hybrid

Exterior (Internet):
• BGP: Path vector, policy-based

OSPF Deep Dive

Open Shortest Path First. Link-state, areas for scalability.

  • Hello packets discover neighbors
  • LSA floods topology info
  • SPF calculates best paths

BGP: Internet's Glue

Border Gateway Protocol routes between ISPs. Policy-driven, not shortest path.

AS Numbers: Autonomous Systems Attributes: Local pref, AS path

Routing Metrics

RIP: Hop count (max 15)
OSPF: Cost = 10^8 / bandwidth
BGP: AS path length + policies

Practice: Traceroute Simulation

Path Discovery:

async function tracerouteSim() {
  const hops = ['Your Router', 'ISP Gateway', 'Regional Hub', 'Destination'];
  console.log('Traceroute simulation:');
  
  for (let hop = 1; hop <= hops.length; hop++) {
    // Simulate latency
    await new Promise(r => setTimeout(r, 50 * hop));
    console.log(`${hop.toString().padStart(2)}: ${hops[hop-1]} (${50*hop}ms)`);
  }
  
  console.log('Trace complete!');
}
tracerouteSim();

Key Takeaways

  • Static: Manual routes
  • OSPF: Link-state IGP (SPF calc)
  • BGP: EGP for Internet
  • RIP: Hop count (simple)
  • Dynamic > Static for large networks
