🌐 Lesson 10: HTTP/HTTPS Protocols

Web protocols, methods, status codes, TLS handshake, proxies, load balancing

HTTP Request/Response

Stateless application protocol (TCP port 80/443).

Request Format:

GET / HTTP/1.1
Host: example.com
User-Agent: Chrome
Accept: text/html

Response: Status + Headers + Body

HTTP Methods

CRUD Methods:

GET:    Read resource
POST:   Create resource  
PUT:    Update (full)
PATCH:  Update (partial)
DELETE: Remove
HEAD:   Metadata only

Status Codes

2xx Success 3xx Redirect 4xx Client Error 5xx Server Error 
200 OK
301 Moved Permanently
404 Not Found
500 Internal Server Error
502 Bad Gateway

HTTPS & TLS

TLS 1.3 encrypts HTTP. Handshake: ClientHello → ServerHello → Key exchange.

HSTS forces HTTPS. Let's Encrypt free certs.

Proxies & Load Balancers

Proxy: Intermediary server. Load balancer: Distributes traffic.

  • Reverse proxy: Protects backend
  • Forward proxy: Client anonymity
  • Layer 4/7 balancing

HTTP/2 & HTTP/3

HTTP/2: Multiplexing, header compression (TCP).
HTTP/3: QUIC over UDP (faster, reliable).

Practice: HTTP Client

Advanced Fetch:

async function httpTest() {
  const req = {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({message: 'Hello HTTP!'})
  };
  
  try {
    const res = await fetch('https://httpbin.org/post', req);
    const data = await res.json();
    
    console.log(`Status: ${res.status} ${res.statusText}`);
    console.log('Echo:', data.json.message);
    console.log('Headers:', Object.fromEntries(res.headers));
  } catch(e) {
    console.log('Error:', e.message);
  }
}
httpTest();

Key Takeaways

  • HTTP stateless: Request/Response cycle
  • GET/POST/PUT/DELETE methods
  • 2xx good, 4xx client, 5xx server
  • HTTPS = HTTP + TLS encryption
  • Reverse proxy hides backend servers
