🔒 Lesson 8: Security Basics

Protecting your computer and data

Why Security Matters

In our connected world, computer security is essential. Every day, cybercriminals attempt to steal data, money, and identities. Understanding security basics helps protect yourself and others.

The reality: There are malicious actors constantly scanning the Internet for vulnerable systems. A new computer connected to the Internet can be attacked within minutes.

Common Threats

Malware (Malicious Software)

Software designed to harm or exploit systems. Types include:

  • Virus: Attaches to files, spreads when files are shared
  • Worm: Self-replicates across networks
  • Trojan: Disguises as legitimate software
  • Ransomware: Encrypts your files, demands payment
  • Spyware: Secretly monitors your activity

Phishing

Fake emails or websites that trick you into revealing passwords or personal information. They often impersonate banks, social media, or trusted companies.

// Example phishing email signs:

From: "Bank Security" 
// Suspicious domain (not your real bank)

Subject: URGENT: Your account will be closed!
// Creates panic to bypass critical thinking

Link: http://bank-verify.com/login
// Not the real bank website!

Request: "Click here to verify your password"
// Banks never ask for passwords via email

Social Engineering

Manipulating people into revealing information or performing actions. The human element is often the weakest link in security.

Man-in-the-Middle Attacks

An attacker intercepts communication between two parties, potentially reading or modifying the data.

Password Security

Passwords are your first line of defense. Strong password practices are crucial:

Creating Strong Passwords

// Password Strength Examples:

WEAK    → "password123"      // Common, easily guessed
WEAK    → "john1990"          // Personal info
MEDIUM  → "P@ssw0rd!"         // Common with substitutions
STRONG  → "Xk9#mP2$vL5@nQ8!"  // Random, complex
STRONG  → "correct-horse-battery-staple" // Long passphrase

Best Practice: Use a password manager! It generates and stores unique, complex passwords for each account. You only need to remember one master password.

Two-Factor Authentication (2FA)

Requires something you know (password) AND something you have (phone, security key). Even if someone steals your password, they can't access your account without the second factor.

// 2FA Types:

SMS Code     → Texted to your phone
Auth App     → Google/Microsoft Authenticator
Hardware Key → Physical device (YubiKey)
Biometric    → Fingerprint, face recognition

Encryption

Encryption converts data into a coded form that can only be read with the correct key. It protects your data both in transit and at rest.

How Encryption Works

// Simplified Encryption:

Original: "Hello World"
    ↓
Encryption (using key)
    ↓
Encrypted: "U2FsdGVkX1+abc123..."
    ↓
Decryption (using same key)
    ↓
Original: "Hello World"

Encryption Types

  • Symmetric: Same key encrypts and decrypts (faster, but key must be shared securely)
  • Asymmetric: Two keys - public (encrypt) and private (decrypt). Used in HTTPS.

Where You See Encryption

  • HTTPS: Encrypts web traffic (look for the padlock!)
  • End-to-end encryption: Messaging apps like Signal, WhatsApp
  • Full disk encryption: Protects data if device is stolen

Safe Browsing Practices

Recognizing Secure Websites

  • Look for "https://" (not just "http://")
  • Check for the padlock icon in the address bar
  • Verify the domain name is correct (no typos!)
  • Be wary of shortened URLs (bit.ly, etc.)

Download Safety

  • Only download from official sources
  • Be cautious of "free" software - it may include malware
  • Scan downloaded files with antivirus software
  • Don't open unexpected email attachments

Public WiFi Risks

Public WiFi networks (coffee shops, airports) are often unsecured. Others on the network could potentially see your traffic.

Public WiFi Safety: Use a VPN (Virtual Private Network) on public WiFi. It creates an encrypted tunnel for your traffic, keeping it private from others on the network.

Keeping Software Updated

Software updates often include security patches for newly discovered vulnerabilities.

// Why updates matter:

1. Vulnerability discovered → Hackers start exploiting it
2. Developer creates patch   → Fixes the security hole
3. Update released          → Available to users
4. You install update       → Now protected!

// If you don't update, you remain vulnerable!

What to Update

  • Operating System: Enable automatic updates
  • Browsers: Usually auto-update, but check
  • Applications: Especially security software
  • Antivirus: Must update for new threat definitions

Backup and Recovery

Security isn't just about prevention - it's also about being able to recover if something goes wrong.

The 3-2-1 Backup Rule

// 3-2-1 Backup Strategy:

3 copies of your data
2 different storage types
1 copy offsite (cloud or different location)

// Example:
- Original on your computer
- Backup on external hard drive
- Backup in cloud storage (Google Drive, Dropbox)

This ensures that even if your computer is lost, stolen, or hit by ransomware, you can recover your data.

Key Takeaways

Remember:

  • Threats: Malware, phishing, social engineering
  • Passwords: Use strong, unique passwords and a password manager
  • 2FA: Enable two-factor authentication everywhere possible
  • Encryption: Protects data in transit and at rest
  • Updates: Keep all software updated for security patches
  • Backups: Follow the 3-2-1 rule for data protection
