📦 Lesson 5: How Software Works

Applications, programs, and execution

What is Software?

Software is a collection of instructions that tells the computer what to do. Unlike hardware (physical components), software is intangible - it's the code that makes hardware useful.

Simple definition: Hardware = the physical machine. Software = the instructions that tell the machine what to do.

Types of Software

System Software

Software that manages computer hardware and provides a platform for running applications.

  • Operating Systems: Windows, macOS, Linux
  • Device Drivers: Software that controls hardware devices
  • Utilities: Antivirus, disk cleanup, backup tools

Application Software

Programs designed for end-users to perform specific tasks.

  • Productivity: Microsoft Word, Excel, Google Docs
  • Communication: Email clients, messaging apps
  • Entertainment: Games, media players
  • Creative: Photoshop, video editors

Programming Software

Tools used to create other software.

  • IDEs: Integrated Development Environments (VS Code, PyCharm)
  • Compilers: Convert source code to machine code
  • Debuggers: Help find and fix errors in code

How Programs are Created

Software development involves writing code in programming languages that humans can understand, then converting it to a form computers can execute.

The Programming Process

// Software Development Process:

1. Write Source Code
    ↓   // Human-readable instructions
    
2. Compile/Interpret
    ↓   // Convert to machine-readable format
    
3. Execute
    ↓   // Computer runs the instructions
    
4. Output
        // Results shown to user

Source Code vs Machine Code

Source Code

Instructions written by programmers in a programming language like Python, JavaScript, or C++. It's human-readable but computers can't understand it directly.

// Example source code in Python:
def greet(name):
    return f"Hello, {name}!"

message = greet("World")
print(message)

Machine Code

The binary instructions that the CPU actually executes. Each CPU architecture has its own machine code format.

// Same program in machine code (simplified):
10110000 01100001
10111000 00000100
11000110 00000111
01010110 10111000
// ... many more lines of binary

Imagine: If programmers had to write machine code directly, even simple programs would take months and be nearly impossible to debug!

Compilers vs Interpreters

There are two main ways to convert source code to machine code:

Compiled Languages

A compiler translates the entire program into machine code before execution. The result is a standalone executable file.

// Compiled Language Workflow:
Source Code → [Compiler] → Executable → [Run]

// Examples: C, C++, Rust, Go
// Pros: Fast execution, optimized code
// Cons: Must recompile after changes, platform-specific

Interpreted Languages

An interpreter reads and executes code line by line, translating on the fly.

// Interpreted Language Workflow:
Source Code → [Interpreter] → Direct Execution

// Examples: Python, JavaScript, Ruby
// Pros: Easier to test, platform-independent
// Cons: Slower execution, needs interpreter installed

Hybrid Approach (JIT)

Modern languages often use Just-In-Time compilation, combining benefits of both approaches. The code is interpreted initially but frequently-used parts are compiled for speed.

  • JavaScript in modern browsers
  • Java (compiles to bytecode, then JIT)
  • Python with PyPy

How Programs Use Memory

When a program runs, it uses different types of memory:

Stack Memory

Fast, organized memory for function calls and local variables. Automatically managed - variables are "popped off" when functions end.

Heap Memory

Larger, flexible memory for dynamic data. Must be managed (manually in some languages, automatically with garbage collection in others).

// Memory usage example (Python):

def calculate():
    x = 10        // Stored on stack
    y = 20        // Stored on stack
    return x + y

// When function ends, x and y are removed from stack

data = [1, 2, 3, 4, 5]  // List stored on heap
// List persists until no longer needed, then garbage collected

APIs and Libraries

Programmers don't write everything from scratch. They use pre-built code:

Libraries

Collections of pre-written code that you can use in your programs. They provide common functionality so you don't have to reinvent the wheel.

APIs (Application Programming Interfaces)

Defined ways for software components to communicate. APIs let you use functionality from other programs or services.

// Example: Using a library in Python
import random  // Import the random library

// Use a function from the library
number = random.randint(1, 100)
print(number)  // Prints random number 1-100

// Without the library, you'd have to write
// all the random number generation code yourself!

Key Takeaways

Remember:

  • Software: Instructions that tell hardware what to do
  • Source Code: Human-readable programming instructions
  • Machine Code: Binary instructions CPUs understand
  • Compilers: Convert entire programs before execution
  • Interpreters: Execute code line by line
  • Libraries/APIs: Pre-built code you can reuse
