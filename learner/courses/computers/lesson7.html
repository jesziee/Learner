🌍 Lesson 7: The Internet and Web

Understanding the connected world

Internet vs World Wide Web

Many people use these terms interchangeably, but they're different things!

The Internet

The Internet is the physical infrastructure - millions of interconnected computers, cables, routers, and servers forming a global network. Think of it as the road system.

The World Wide Web (WWW)

The Web is a service that runs ON the Internet - a collection of websites and web pages linked together. Think of it as the vehicles and destinations on the roads.

Simple comparison: The Internet is like the telephone network infrastructure. The Web is like one of the services that uses it (like calling a friend). Email, video streaming, and online gaming also use the Internet but aren't part of the Web.

Key distinction: The Internet was created in the 1960s-70s. The World Wide Web was invented in 1989 by Tim Berners-Lee.

How the Web Works

When you visit a website, a complex process happens in seconds:

// The Web Request Process:

1. You type: https://www.example.com
    ↓
2. DNS Lookup: Find the server's IP address
    ↓
3. HTTP Request: Browser sends request to server
    ↓
4. Server Processing: Server finds the webpage
    ↓
5. HTTP Response: Server sends files back
    ↓
6. Rendering: Browser displays the webpage

Client-Server Model

The web operates on a client-server model:

  • Client: Your web browser (Chrome, Firefox, Safari) - it requests and displays content
  • Server: A computer that stores websites and sends them to clients when requested

Web Technologies

HTML (HyperText Markup Language)

The structure of web pages. HTML defines what content appears on a page - headings, paragraphs, images, links.

CSS (Cascading Style Sheets)

The styling of web pages. CSS controls how content looks - colors, fonts, layouts, animations.

JavaScript

The behavior of web pages. JavaScript makes pages interactive - responding to clicks, loading new content, validating forms.

// How these technologies work together:

HTML → "What's on the page"  // Structure
CSS  → "How it looks"      // Presentation
JS   → "How it behaves"    // Behavior

// Analogy:
// HTML = skeleton (structure)
// CSS = skin and clothes (appearance)
// JavaScript = muscles and movement (behavior)

URLs Explained

A URL (Uniform Resource Locator) is the address of a web resource. Let's break one down:

// URL Structure:
https://www.example.com/page?id=123#section

// Breakdown:
┌──────────────────────────────────────────────────────┐
│ https      → Protocol (how to communicate)       │
│ www        → Subdomain (specific server)           │
│ example    → Domain name (website identity)       │
│ com        → Top-level domain (TLD)              │
│ /page     → Path (specific page)                   │
│ ?id=123   → Query parameters (data sent to server)│
│ #section  → Fragment (jump to section on page)    │
└──────────────────────────────────────────────────────┘

Common TLDs

  • .com: Commercial (most common)
  • .org: Organizations
  • .edu: Educational institutions
  • .gov: Government
  • .io: Tech startups (originally British Indian Ocean Territory)
  • .country codes: .uk, .ca, .de, .jp, etc.

HTTP and HTTPS

HTTP (HyperText Transfer Protocol)

The rules for transferring web pages. It defines how clients and servers communicate.

HTTPS (HTTP Secure)

Encrypted HTTP. It protects your data from being intercepted. Always look for the padlock icon in your browser!

// HTTP Methods (types of requests):

GET     → "Give me this page"
POST    → "Here's data to process"
PUT     → "Update this resource"
DELETE  → "Remove this resource"

// HTTP Status Codes:
200 → OK (success)
301 → Moved Permanently (redirect)
404 → Not Found (page doesn't exist)
500 → Internal Server Error (server problem)

Security Tip: Never enter passwords or credit card info on non-HTTPS sites. Look for the padlock icon and "https://" in the URL.

Web Browsers

Browsers are software that fetch, interpret, and display web pages. Popular browsers include:

  • Google Chrome: Most popular, fast, great developer tools
  • Mozilla Firefox: Privacy-focused, open source
  • Safari: Apple's browser, optimized for macOS and iOS
  • Microsoft Edge: Built on Chromium, integrated with Windows

What Browsers Do

  1. Parse HTML to build the page structure
  2. Apply CSS to style elements
  3. Execute JavaScript for interactivity
  4. Handle user interactions (clicks, forms)
  5. Cache resources for faster loading

Frontend vs Backend

Frontend (Client-side)

Everything the user sees and interacts with in the browser. Built with HTML, CSS, and JavaScript.

Backend (Server-side)

The server-side logic that powers applications. Handles databases, authentication, and business logic. Built with languages like Python, Node.js, PHP, Ruby, or Java.

// Web Application Architecture:

┌─────────────────────────────────────────┐
│              FRONTEND                  │
│  HTML + CSS + JavaScript (in browser)   │
└─────────────────┬───────────────────────┘
                  │ HTTP requests
                  ↓
┌─────────────────────────────────────────┐
│              BACKEND                   │
│  Server (Python/Node.js/etc)            │
│  ↓                                      │
│  Database (MySQL/MongoDB/etc)           │
└─────────────────────────────────────────┘

Key Takeaways

Remember:

  • Internet: Global network infrastructure
  • Web: Service running on the Internet
  • HTML/CSS/JS: The three core web technologies
  • URL: Address that identifies web resources
  • HTTPS: Secure, encrypted web communication
  • Frontend: User-facing; Backend: Server-side
