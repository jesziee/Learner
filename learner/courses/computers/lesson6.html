🌐 Lesson 6: Computer Networks Basics

How computers communicate with each other

What is a Computer Network?

A computer network is a group of computers connected together to share resources and communicate. Networks range from small home networks to the global Internet.

Why Networks Matter: Without networks, each computer would be isolated. Networks enable email, web browsing, video calls, online gaming, and virtually everything we do with computers today.

Types of Networks

PAN (Personal Area Network)

Very short-range network around a person. Example: Bluetooth connections between your phone and headphones.

LAN (Local Area Network)

Network in a limited area like a home, office, or building. Your home WiFi network is a LAN.

WAN (Wide Area Network)

Network spanning large geographic areas. The Internet is the largest WAN.

WLAN (Wireless LAN)

A LAN that uses wireless communication (WiFi) instead of cables.

Key Point: The Internet is a network of networks. Your home network connects to your ISP's network, which connects to other networks worldwide.

Network Hardware

Router

Directs data between networks. Your home router connects your LAN to the Internet (WAN) and decides where to send data.

Switch

Connects devices within a LAN. Unlike a router, it only handles local traffic between connected devices.

Modem

Converts signals between your ISP's network and your home network. Often combined with routers in "gateway" devices.

Network Interface Card (NIC)

Hardware that lets a computer connect to a network. Most computers have built-in WiFi and Ethernet NICs.

// Simplified network structure:

Internet
    ↓
Modem (converts ISP signals)
    ↓
Router (directs traffic, provides WiFi)
    ↓
Switch (optional, expands wired connections)
    ↓
Devices (computers, phones, tablets)

IP Addresses

Every device on a network needs a unique address - like a postal address for your computer. This is called an IP (Internet Protocol) address.

IPv4 Addresses

The traditional format: four numbers separated by dots. Each number ranges from 0-255.

// IPv4 Address Example:
192.168.1.1

// Breakdown:
192.168.1.1
 ↑    ↑   ↑   ↑
0-255 for each section

// Total possible IPv4 addresses:
256 × 256 × 256 × 256 ≈ 4.3 billion

IPv6 Addresses

With 4.3 billion IPv4 addresses running out, IPv6 was created with a much larger address space.

// IPv6 Address Example:
2001:0db8:85a3:0000:0000:8a2e:0370:7334

// Uses hexadecimal (0-9, a-f)
// 128 bits vs IPv4's 32 bits
// Approximately 340 undecillion addresses!

Public vs Private IPs

  • Public IP: Unique address visible on the Internet. Assigned by your ISP.
  • Private IP: Used within local networks. Not directly accessible from the Internet.
// Common private IP ranges:
192.168.0.0 - 192.168.255.255  // Home routers
10.0.0.0 - 10.255.255.255    // Large organizations
172.16.0.0 - 172.31.255.255  // Medium networks

Ports and Protocols

What are Ports?

Ports are numbered channels that help route data to the right application on a device. Think of an IP address as a building address and ports as apartment numbers.

// Common port numbers:
80   → HTTP   (web browsing)
443  → HTTPS  (secure web browsing)
22   → SSH    (secure remote access)
25   → SMTP   (email sending)
53   → DNS    (domain name system)
3306 → MySQL  (database)

// Example: https://example.com:443
// The :443 specifies the port (usually hidden for standard ports)

Common Protocols

Protocols are standardized rules for communication:

  • HTTP/HTTPS: Transfers web pages
  • FTP: File transfers
  • SMTP/POP/IMAP: Email
  • DNS: Translates domain names to IP addresses
  • TCP: Reliable, connection-based data transfer
  • UDP: Fast, connectionless data transfer (used for streaming)

TCP vs UDP: TCP guarantees data arrives correctly (like certified mail). UDP is faster but doesn't guarantee delivery (like regular mail). Video streaming uses UDP because speed matters more than perfection.

DNS (Domain Name System)

DNS translates human-readable domain names to IP addresses. It's like the phonebook of the Internet.

// DNS Translation Example:

You type: www.google.com
        ↓
DNS Lookup (finds the IP address)
        ↓
Browser connects to: 142.250.190.46

// Without DNS, you'd have to memorize
// IP addresses for every website!

How DNS Works

  1. You type a domain name in your browser
  2. Your computer checks its local cache
  3. If not found, it asks your ISP's DNS server
  4. If still not found, it asks root DNS servers
  5. The chain continues until the IP is found
  6. The result is cached for future use

Key Takeaways

Remember:

  • Network: Connected computers sharing resources
  • LAN: Local network; WAN: Wide-area network
  • Router: Directs traffic between networks
  • IP Address: Unique identifier for each device
  • Ports: Channels for routing data to applications
  • DNS: Translates domain names to IP addresses
