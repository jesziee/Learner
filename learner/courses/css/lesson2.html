Lesson 2: Selectors

Learn how to target HTML elements for styling

Lesson 2 of 12

What are Selectors?

Selectors are patterns that tell CSS which HTML elements to style. Without selectors, CSS wouldn't know what to apply styles to. Mastering selectors is essential for writing efficient, targeted CSS. Think of selectors as the "address" that tells CSS where to deliver the styles.

1. Element (Type) Selectors

The simplest selector — targets all elements of a specific type.

/* Element Selector: targets all paragraphs */
p {
  color: #333;
  line-height: 1.6;
}

/* Targets all headings */
h1 {
  font-size: 32px;
  color: #7C3AED;
}

/* Targets all links */
a {
  color: purple;
  text-decoration: none;
}

/* Targets all images */
img {
  max-width: 100%;
  height: auto;
}
How it works:
  • Use the HTML tag name without brackets: p, h1, div
  • Styles apply to ALL elements of that type on the page
  • Good for base styles that should be consistent

2. Class Selectors

Target elements with a specific class attribute. Classes are the most versatile selectors.

/* HTML */

This paragraph is highlighted.


This paragraph is normal.


Another highlighted paragraph.


/* CSS */
/* Class selector uses a dot (.) before the class name */
.highlight {
  background-color: yellow;
  padding: 2px 5px;
}

/* Multiple classes on one element */


/* Target specific combination */
.btn-primary {
  background-color: #7C3AED;
  color: white;
}

.btn-large {
  padding: 15px 30px;
  font-size: 18px;
}
Class selector rules:
  • Syntax: Dot followed by class name: .classname
  • Reusable: Same class can be used on multiple elements
  • Multiple classes: Elements can have multiple classes (space-separated)
  • Naming: Use lowercase, hyphens for spaces: .btn-primary
Class naming best practices:
  • Use descriptive names: .error-message not .red-text
  • Use lowercase with hyphens: .nav-link
  • Avoid starting with numbers: .1button is invalid

3. ID Selectors

Target a single element with a unique ID. IDs should be unique per page.

/* HTML */


  
...






  
...




/* CSS */
/* ID selector uses a hash (#) before the ID name */
#main-header {
  background-color: #7C3AED;
  padding: 20px;
  position: sticky;
  top: 0;
}

#primary-nav {
  display: flex;
  gap: 20px;
}

#featured-post {
  border: 2px solid #7C3AED;
  padding: 20px;
}
ID selector characteristics:
  • Syntax: Hash followed by ID name: #idname
  • Unique: Only one element per page should have each ID
  • High specificity: ID selectors override class selectors
  • Use for: Landmarks, skip link targets, JavaScript hooks
ID vs Class: Prefer classes for styling. IDs are better for JavaScript and page landmarks. ID's high specificity can make CSS harder to maintain.

4. Attribute Selectors

Target elements based on their attributes or attribute values.

/* Target elements with a specific attribute */
[disabled] {
  opacity: 0.5;
  cursor: not-allowed;
}

/* Target elements with specific attribute value */
[type="email"] {
  background-image: url('email-icon.svg');
  background-position: right 10px center;
  background-repeat: no-repeat;
}

[type="password"] {
  letter-spacing: 2px;
}

/* Target links that start with "https" */
[href^="https"] {
  color: green;
}

/* Target links ending with ".pdf" */
[href$=".pdf"] {
  padding-right: 20px;
  background-image: url('pdf-icon.svg');
  background-repeat: no-repeat;
}

/* Target links containing "example" */
[href*="example"] {
  font-weight: bold;
}
Attribute selector patterns:
  • [attr] — Has the attribute
  • [attr="value"] — Attribute equals value exactly
  • [attr^="value"] — Attribute starts with value
  • [attr$="value"] — Attribute ends with value
  • [attr*="value"] — Attribute contains value

5. Descendant and Child Selectors

Target elements based on their relationship to other elements.

/* HTML structure */


  
    
    
  • Home
    • 
    
  • About
    • 
  






  
Paragraph with a link inside.




/* Descendant selector (space) - any nested element */
.main-nav a {
  color: white;
  text-decoration: none;
}
/* Targets ALL links inside .main-nav, at any depth */

/* Child selector (>) - direct children only */
.main-nav > ul {
  list-style: none;
}
/* Targets only ul that is a DIRECT child of .main-nav */

/* Combined selectors */
article p a {
  color: #7C3AED;
  border-bottom: 1px dotted #7C3AED;
}
/* Links inside paragraphs inside articles */
Combinator comparison:
  • .parent .child (space) — Any descendant at any level
  • .parent > .child — Direct child only
  • Descendant is more flexible but can be slower
  • Child is more specific and performs better

6. Pseudo-Class Selectors

Target elements based on state or position. Use a single colon (:).

/* Mouse interaction states */
a:link {
  color: blue;              /* Unvisited link */
}

a:visited {
  color: purple;            /* Visited link */
}

a:hover {
  color: #7C3AED;           /* Mouse over */
  text-decoration: underline;
}

a:active {
  color: red;               /* Being clicked */
}

a:focus {
  outline: 2px solid #7C3AED; /* Keyboard focus */
}

/* Form states */
input:focus {
  border-color: #7C3AED;
  box-shadow: 0 0 5px rgba(124, 58, 237, 0.3);
}

input:disabled {
  background-color: #f0f0f0;
  cursor: not-allowed;
}

input:valid {
  border-color: green;
}

input:invalid {
  border-color: red;
}

/* Position-based */
li:first-child {
  font-weight: bold;
}

li:last-child {
  border-bottom: none;
}

li:nth-child(odd) {
  background-color: #f9f9f9;
}

li:nth-child(3) {
  color: purple;            /* Third item only */
}
Common pseudo-classes:
  • :hover — Mouse is over element
  • :focus — Element has keyboard focus
  • :active — Element is being activated (clicked)
  • :first-child — First element among siblings
  • :last-child — Last element among siblings
  • :nth-child(n) — nth element (odd, even, 3, 2n+1)

7. Pseudo-Element Selectors

Target specific parts of an element. Use double colon (::).

/* First letter of text */
p::first-letter {
  font-size: 2em;
  font-weight: bold;
  color: #7C3AED;
  float: left;
  margin-right: 5px;
}

/* First line of text */
p::first-line {
  font-weight: bold;
  color: #333;
}

/* Insert content before an element */
.required::before {
  content: "* ";
  color: red;
}

/* Insert content after an element */
.new-link::after {
  content: " (new)";
  color: #7C3AED;
  font-size: 0.8em;
}

/* Style placeholder text */
input::placeholder {
  color: #999;
  font-style: italic;
}

/* Style selected text */
::selection {
  background-color: #7C3AED;
  color: white;
}
Pseudo-element uses:
  • ::before / ::after — Add decorative content
  • ::first-letter — Drop caps, styling initials
  • ::first-line — Style first line differently
  • ::placeholder — Style form placeholders
  • ::selection — Style highlighted text

8. Grouping Selectors

Apply the same styles to multiple selectors using commas.

/* Group multiple selectors with commas */
h1, h2, h3, h4, h5, h6 {
  color: #333;
  font-family: 'Georgia', serif;
  margin-top: 0;
}

/* Mix different selector types */
.highlight, .featured, #top-story {
  border-left: 4px solid #7C3AED;
  padding-left: 15px;
}

/* Group pseudo-classes */
a:hover, a:focus {
  color: #9333EA;
  text-decoration: underline;
}

/* Combine with descendant selectors */
.main-nav a, .footer-nav a, .sidebar a {
  text-decoration: none;
}
Tip: Grouping selectors reduces code duplication and keeps your CSS DRY (Don't Repeat Yourself).

9. Specificity: Which Selector Wins?

When multiple selectors target the same element, specificity determines which styles apply.

/* Specificity ranking (lowest to highest): */

/* 1. Element selectors (specificity: 0-0-1) */
p { color: black; }

/* 2. Class selectors (specificity: 0-1-0) */
.text { color: blue; }

/* 3. ID selectors (specificity: 1-0-0) */
#intro { color: green; }

/* 4. Inline styles (specificity: 1-0-0-0) */

Text


/* Calculating specificity */
p { }                    /* 0-0-1 = 1 point */
.text { }                /* 0-1-0 = 10 points */
p.text { }               /* 0-1-1 = 11 points */
#intro { }               /* 1-0-0 = 100 points */
#intro.text { }          /* 1-1-0 = 110 points */

/* Example conflict */
p { color: black; }      /* Loses to class */
.text { color: blue; }   /* Wins */

I'm blue!
Specificity rules:
  • Inline styles beat ID selectors
  • ID selectors beat class selectors
  • Class selectors beat element selectors
  • More specific selectors override less specific ones
  • When equal specificity, last declared wins

📚 Key Takeaways

  • Element selectors target all elements of a type (p, h1, div)
  • Class selectors (.class) are reusable and versatile
  • ID selectors (#id) should be unique per page
  • Attribute selectors target elements by attributes
  • Descendant (space) and child (>) selectors target nested elements
  • Pseudo-classes (:) target states (:hover, :focus)
  • Pseudo-elements (::) target parts of elements (::before, ::after)
  • Specificity determines which styles win when conflicts occur
