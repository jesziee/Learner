Lesson 4: Box Model

Understanding how CSS treats every element as a box

Lesson 4 of 12

The CSS Box Model

In CSS, every element is treated as a rectangular box. Understanding the box model is essential for controlling layout, spacing, and sizing. The box model consists of four areas: content, padding, border, and margin. These layers determine how much space an element takes up and how it relates to other elements.

The Four Layers of the Box Model

Every element box has four concentric layers:

/* Box Model Visualization */
/*
┌─────────────────────────────────────┐
│              MARGIN                 │  ← Outermost: Space outside
│   ┌─────────────────────────────┐   │
│   │           BORDER            │   │  ← The border edge
│   │   ┌─────────────────────┐   │   │
│   │   │      PADDING        │   │   │  ← Space inside border
│   │   │   ┌─────────────┐   │   │   │
│   │   │   │   CONTENT   │   │   │   │  ← The actual content
│   │   │   └─────────────┘   │   │   │
│   │   └─────────────────────┘   │   │
│   └─────────────────────────────┘   │
└─────────────────────────────────────┘
*/
The four layers:
  • Content: The actual content (text, images)
  • Padding: Space between content and border
  • Border: The edge around the padding
  • Margin: Space outside the border, between elements

1. Content Area

The content area holds the element's actual content — text, images, or child elements.

/* Width and Height control the content area */
.box {
  width: 300px;           /* Content width */
  height: 200px;          /* Content height */
}

/* Min and Max dimensions */
.responsive {
  width: 100%;
  max-width: 1200px;      /* Won't exceed 1200px */
  min-width: 300px;       /* Won't go below 300px */
}

.sidebar {
  width: 250px;
  min-height: 100vh;      /* At least full viewport height */
}

/* Intrinsic sizing keywords */
.element {
  width: fit-content;     /* Only as wide as content needs */
  width: min-content;     /* As narrow as possible */
  width: max-content;     /* As wide as content wants */
}
Width keywords:
  • auto — Default, fills available space
  • fit-content — Shrinks to fit content
  • min-content — Minimum width of content
  • max-content — Maximum width of content

2. Padding

Padding creates space between the content and the border. It's inside the element and inherits the element's background.

/* Padding - space inside the border */
.box {
  /* All sides equal */
  padding: 20px;
}

/* Different sides: top right bottom left (clockwise) */
.card {
  padding: 10px 20px 15px 25px;
  /*       top  right bottom left */
}

/* Three values: top, right/left, bottom */
.element {
  padding: 10px 20px 15px;
  /*       top  sides  bottom */
}

/* Two values: top/bottom, right/left */
.section {
  padding: 20px 40px;
  /*       vertical horizontal */
}

/* Individual sides */
.custom {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 10px;
  padding-left: 20px;
}

/* Inline shorthand */
.inline-padding {
  padding-inline: 20px;      /* Left and right in LTR */
  padding-block: 10px;       /* Top and bottom */
}
Padding shorthand patterns:
  • padding: 20px; — All four sides
  • padding: 10px 20px; — Vertical | Horizontal
  • padding: 10px 20px 15px; — Top | Sides | Bottom
  • padding: 10px 20px 15px 25px; — Top | Right | Bottom | Left
Remember: Padding increases the total size of the element (by default). A 300px wide box with 20px padding is actually 340px wide!

3. Border

The border sits between padding and margin. It can be visible (styled) or invisible but still takes space.

/* Border shorthand: width style color */
.box {
  border: 2px solid #7C3AED;
}

/* Individual properties */
.custom-border {
  border-width: 2px;
  border-style: solid;
  border-color: #7C3AED;
}

/* Border styles */
.style-examples {
  border-style: solid;      /* Solid line */
  border-style: dashed;     /* Dashed line */
  border-style: dotted;     /* Dotted line */
  border-style: double;     /* Double line */
  border-style: groove;     /* 3D grooved */
  border-style: ridge;      /* 3D ridged */
  border-style: inset;      /* 3D inset */
  border-style: outset;     /* 3D outset */
  border-style: none;       /* No border */
  border-style: hidden;     /* Hidden border */
}

/* Individual sides */
.asymmetric {
  border-top: 3px solid #7C3AED;
  border-right: 1px dashed #ccc;
  border-bottom: 2px dotted #999;
  border-left: none;
}

/* Border radius (rounded corners) */
.rounded {
  border-radius: 8px;              /* All corners */
}

.circle {
  border-radius: 50%;              /* Perfect circle (if square) */
}

.pill {
  border-radius: 100px;            /* Pill shape */
}

/* Individual corners */
.custom-radius {
  border-radius: 10px 20px 10px 20px;
  /*            top-left top-right bottom-right bottom-left */
}

/* Elliptical corners */
.ellipse {
  border-radius: 20px / 10px;      /* Horizontal / Vertical radius */
}
Border properties:
  • border-width — Thickness (px, em, thin, medium, thick)
  • border-style — solid, dashed, dotted, double, none
  • border-color — Any valid color
  • border-radius — Rounds the corners

4. Margin

Margin creates space outside the border, separating elements from each other. Margins are transparent.

/* Margin - space outside the border */
.box {
  /* All sides equal */
  margin: 20px;
}

/* Same shorthand patterns as padding */
.card {
  margin: 10px 20px 15px 25px;   /* Top Right Bottom Left */
  margin: 10px 20px;              /* Vertical Horizontal */
  margin: 10px 20px 15px;         /* Top Sides Bottom */
}

/* Individual sides */
.custom-margin {
  margin-top: 10px;
  margin-right: 20px;
  margin-bottom: 15px;
  margin-left: 25px;
}

/* Auto margins for centering */
.centered {
  width: 300px;
  margin-left: auto;
  margin-right: auto;
  /* OR shorthand */
  margin: 0 auto;
}

/* Horizontal centering */
.container {
  max-width: 1200px;
  margin: 0 auto;        /* Center horizontally */
}

/* Inline/Block margins */
.horizontal-space {
  margin-inline: auto;   /* Left/right in LTR */
  margin-block: 20px;    /* Top/bottom */
}

/* Negative margins (overlap) */
.overlap {
  margin-top: -20px;     /* Pulls element up */
}

.pull-left {
  margin-left: -10px;    /* Extends beyond container */
}
Margin behaviors:
  • margin: auto — Takes available space, used for centering
  • Negative margins — Pull elements closer or extend beyond
  • Percentages — Relative to parent's width

5. Margin Collapsing

Vertical margins between block elements collapse into a single margin equal to the larger one.

/* Margin Collapsing Example */

/* HTML */

Box 1


Box 2


/* CSS */
.box-one {
  margin-bottom: 30px;
}

.box-two {
  margin-top: 20px;
}

/* Total space between them: 30px, NOT 50px! */
/* The margins collapse to the larger value */

/* Preventing margin collapse */
.no-collapse-parent {
  overflow: hidden;       /* Creates BFC */
  /* OR */
  display: flow-root;     /* Modern way to create BFC */
}

.no-collapse-child {
  display: inline-block;  /* Inline-blocks don't collapse */
}

/* Margins DON'T collapse when: */
/* - Using Flexbox or Grid */
/* - Parent has overflow: hidden/auto/scroll */
/* - Parent has border or padding */
/* - Elements are positioned or floated */
Margin collapse only happens: Between vertical margins of block elements in normal flow. Horizontal margins never collapse.

6. Box Sizing

The box-sizing property controls how total width/height is calculated.

/* content-box (default) */
.content-box {
  box-sizing: content-box;    /* Default */
  width: 300px;
  padding: 20px;
  border: 5px solid black;
  /* Total width: 300 + 40 + 10 = 350px */
}

/* border-box (recommended) */
.border-box {
  box-sizing: border-box;     /* Width includes padding & border */
  width: 300px;
  padding: 20px;
  border: 5px solid black;
  /* Total width: 300px (padding and border are INSIDE) */
  /* Content area: 300 - 40 - 10 = 250px */
}

/* Best practice: Apply border-box to all elements */
*, *::before, *::after {
  box-sizing: border-box;
}

/* Or inherit from html */
html {
  box-sizing: border-box;
}
*, *::before, *::after {
  box-sizing: inherit;
}
Why border-box is better:
  • Width you set is the total width
  • Easier to calculate layouts
  • Adding padding/border doesn't break layout
  • Most CSS frameworks expect this

7. Display and Box Type

The display property affects how the box model works for an element.

/* Block elements */
div, p, h1, section {
  /* display: block (default) */
  /* - Take full width available */
  /* - Start on new line */
  /* - Width, height, margins all work */
  /* - Can contain other block and inline elements */
}

/* Inline elements */
span, a, strong {
  /* display: inline (default) */
  /* - Only take needed width */
  /* - Don't start new line */
  /* - Width and height ignored */
  /* - Vertical margins/padding work differently */
}

/* Inline-block */
.button {
  display: inline-block;
  /* - Flows like inline */
  /* - But respects width, height, margins like block */
}

/* None */
.hidden {
  display: none;
  /* - Removes element from layout */
  /* - No space reserved */
}

8. Outline vs Border

Outline is similar to border but doesn't affect layout.

/* Outline - doesn't add to box size */
:focus {
  outline: 2px solid #7C3AED;
  outline-offset: 2px;       /* Space between element and outline */
}

/* Differences between border and outline: */
/* 
   BORDER:
   - Part of the box model
   - Takes up space
   - Can have different sides
   - Affects layout
   
   OUTLINE:
   - NOT part of box model
   - Doesn't take space
   - Always same on all sides
   - Doesn't affect layout
   - Often used for focus states
*/

/* Remove default outline (add your own focus style!) */
:focus {
  outline: none;             /* Never do this without adding focus styles! */
  box-shadow: 0 0 0 3px rgba(124, 58, 237, 0.5);
}

9. Box Model Debugging

Browser DevTools make it easy to visualize the box model.

/* How to debug box model issues: */

/* 1. Right-click element → Inspect */
/* 2. Look at the "Computed" panel */
/* 3. See visual diagram of box model */
/* 4. Hover over elements to see box highlighted */

/* Common issues: */

/* Element wider than expected */
.too-wide {
  box-sizing: border-box;    /* Fix: use border-box */
  width: 100%;
  padding: 20px;
}

/* Unexpected spacing */
.weird-space {
  /* Check for margin collapse */
  /* Add overflow: hidden to parent */
}

/* Element not centering */
.not-centering {
  /* Must have a width */
  width: 300px;
  margin: 0 auto;
}

📚 Key Takeaways

  • Every element is a box with content, padding, border, and margin
  • Padding is inside, margin is outside the border
  • Use box-sizing: border-box for predictable sizing
  • Margin collapse affects vertical margins between block elements
  • margin: auto centers block elements horizontally
  • Outline doesn't affect layout, great for focus states
  • Use DevTools to debug box model issues
