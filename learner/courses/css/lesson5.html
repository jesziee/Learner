Understanding the Display Property

The display property is one of the most important CSS properties. It determines how an element is treated in the layout — whether it starts on a new line, how width and height work, and how it interacts with other elements. Mastering display is essential for creating any layout.

1. Block Display

Block elements take up the full width available and always start on a new line.

/* Block elements */
div, p, h1-h6, section, article, header, footer, nav {
  display: block;    /* Default for these elements */
}

/* Making an element block */
span {
  display: block;
}

/* Block element characteristics: */
.block-box {
  display: block;
  
  /* ✓ Starts on a new line */
  /* ✓ Takes full width available (width: auto) */
  /* ✓ Width and height properties work */
  /* ✓ Padding, margin, border work on all sides */
  /* ✓ Can contain other block or inline elements */
}

/* Example: Three divs stacked vertically */
.container div {
  display: block;
  margin-bottom: 10px;
}
/* Result:
   ┌─────────────────┐
   │     Div 1       │
   └─────────────────┘
   ┌─────────────────┐
   │     Div 2       │
   └─────────────────┘
   ┌─────────────────┐
   │     Div 3       │
   └─────────────────┘
*/
Block elements:
  • Always start on a new line
  • Expand to fill container width
  • Can set width, height, margins
  • Examples: div, p, h1, section, article

2. Inline Display

Inline elements flow within text and only take up as much width as necessary.

/* Inline elements */
span, a, strong, em, b, i {
  display: inline;    /* Default for these elements */
}

/* Making an element inline */
div {
  display: inline;
}

/* Inline element characteristics: */
.inline-element {
  display: inline;
  
  /* ✗ Does NOT start on a new line */
  /* ✓ Flows with text */
  /* ✗ Width and height properties are IGNORED */
  /* ✗ Vertical margins/padding don't push other elements */
  /* ✓ Horizontal margins/padding work */
}

/* Example: Links within text */
p {
  font-size: 16px;
}

p a {
  display: inline;    /* Links flow with text */
  color: #7C3AED;
  padding: 0 5px;     /* Horizontal padding works */
  background: #f0e6ff;
}

/* HTML: */

This is a paragraph with a link inside it.
Inline elements:
  • Don't break the flow of text
  • Width/height determined by content
  • Can't set width or height
  • Examples: span, a, strong, em
Important: Vertical padding and margins on inline elements don't affect surrounding content — they visually overflow but don't push other elements.

3. Inline-Block Display

Inline-block combines the flow of inline with the box model of block.

/* Inline-block: Best of both worlds */
.button {
  display: inline-block;
  
  /* ✓ Flows like inline (doesn't break line) */
  /* ✓ Width and height work like block */
  /* ✓ Padding, margin, border work on all sides */
  /* ✓ Can set vertical alignment */
}

/* Example: Navigation links */
.nav-link {
  display: inline-block;
  padding: 10px 20px;
  background-color: #7C3AED;
  color: white;
  text-decoration: none;
  margin-right: 5px;
}

/* Result: Buttons sit side-by-side */
/* [Home] [About] [Contact] */

/* Example: Buttons in a row */
.btn {
  display: inline-block;
  padding: 12px 24px;
  border-radius: 5px;
  cursor: pointer;
}

.btn-primary {
  background-color: #7C3AED;
  color: white;
}

/* HTML: */


/* Buttons appear side by side, not stacked */
Inline-block elements:
  • Flow inline but can have dimensions
  • Width and height can be set
  • Respects vertical-align property
  • Commonly used for buttons, navigation

4. None Display

Display none completely removes the element from the layout.

/* Hide element completely */
.hidden {
  display: none;
  /* - Element is removed from document flow */
  /* - Takes up no space */
  /* - Not visible, not accessible to screen readers */
  /* - Children are also hidden */
}

/* Contrast with visibility */
.invisible {
  visibility: hidden;
  /* - Element is invisible */
  /* - STILL takes up space */
  /* - Layout is preserved */
}

/* Contrast with opacity */
.transparent {
  opacity: 0;
  /* - Element is transparent */
  /* - STILL takes up space */
  /* - Can still be interacted with */
  /* - Transition-able */
}

/* Toggle pattern with JavaScript */
.modal {
  display: none;    /* Hidden by default */
}

.modal.active {
  display: block;   /* Shown when active */
}

/* Mobile menu pattern */
.nav-menu {
  display: none;
}

@media (max-width: 768px) {
  .nav-menu.open {
    display: block;
  }
}
Hiding methods comparison:
  • display: none — Removed from layout, no space
  • visibility: hidden — Invisible, space preserved
  • opacity: 0 — Transparent, space preserved, interactive

5. Modern Display Values

CSS has many more display values for specific use cases.

/* Flexbox - modern layout */
.container {
  display: flex;
  /* Children become flex items */
}

.item {
  flex: 1;    /* Grow to fill space */
}

/* Grid - two-dimensional layout */
.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

/* Table display values */
.table-container {
  display: table;
}

.row {
  display: table-row;
}

.cell {
  display: table-cell;
}

/* List item */
.custom-list-item {
  display: list-item;
  list-style-type: disc;
}

/* Flow-root - creates block formatting context */
.bfc {
  display: flow-root;
  /* Contains floats, prevents margin collapse */
}

/* Contents - element acts like it doesn't exist */
.wrapper {
  display: contents;
  /* Element itself doesn't generate a box */
  /* Only its children are rendered */
}

/* Useful for styling without extra wrapper */
.grid-container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

.grid-wrapper {
  display: contents;    /* Doesn't create extra grid cell */
}

6. Two-Value Display Syntax

Modern CSS uses two values: outer and inner display type.

/* Two-value syntax: display: outer inner; */

/* Outer: how element behaves in layout */
/* Inner: how children are laid out */

/* Traditional vs Modern */
.container {
  /* Traditional */
  display: flex;
  
  /* Modern equivalent */
  display: block flex;   /* Block-level flex container */
}

.inline-flex-container {
  /* Traditional */
  display: inline-flex;
  
  /* Modern equivalent */
  display: inline flex;  /* Inline-level flex container */
}

.grid-container {
  display: block grid;   /* Block-level grid container */
}

.inline-grid-container {
  display: inline grid;  /* Inline-level grid container */
}

/* Why two values? */
/* More explicit and flexible */
/* Separates outer and inner behavior */
/* Better describes what's happening */
Note: Two-value syntax is well-supported in modern browsers. Using the single-value shorthand (flex, grid) is still perfectly fine and widely used.

7. Practical Examples

Navigation Menu

/* Horizontal navigation with inline-block */
nav ul {
  list-style: none;
  padding: 0;
  margin: 0;
}

nav li {
  display: inline-block;    /* List items side by side */
}

nav a {
  display: inline-block;    /* Clickable area larger than text */
  padding: 10px 15px;
  text-decoration: none;
  color: #333;
}

nav a:hover {
  background-color: #7C3AED;
  color: white;
}

Card Grid with inline-block

/* Cards in a grid pattern */
.card-grid {
  text-align: center;       /* Center the cards */
}

.card {
  display: inline-block;
  width: 300px;
  margin: 10px;
  padding: 20px;
  text-align: left;         /* Reset text alignment */
  vertical-align: top;      /* Align tops of cards */
  border: 1px solid #ddd;
  border-radius: 8px;
}

Show/Hide Content

/* Accordion pattern */
.accordion-content {
  display: none;
}

.accordion-item.active .accordion-content {
  display: block;
}

/* Tab content */
.tab-panel {
  display: none;
}

.tab-panel.active {
  display: block;
}

/* Responsive hiding */
.sidebar {
  display: block;
}

@media (max-width: 768px) {
  .sidebar {
    display: none;    /* Hide on mobile */
  }
}

8. Converting Elements

You can change the natural display type of any element.

/* Link as button */
a.btn {
  display: inline-block;
  padding: 10px 20px;
  background-color: #7C3AED;
  color: white;
  text-decoration: none;
  border-radius: 5px;
}

/* Span as block container */
span.block-style {
  display: block;
  width: 200px;
  padding: 20px;
}

/* List as horizontal nav */
ul.horizontal-nav {
  list-style: none;
}

ul.horizontal-nav li {
  display: inline-block;
}

/* Div as inline element */
div.inline-box {
  display: inline;
  /* Now flows with text */
}

/* Paragraph as inline code */
p.code-line {
  display: inline;
  font-family: monospace;
  background: #f5f5f5;
}

📚 Key Takeaways

  • Block elements start on new lines and take full width
  • Inline elements flow with text, can't have width/height
  • Inline-block flows inline but respects width/height
  • display: none removes element from layout completely
  • visibility: hidden hides but preserves space
  • Modern display uses flex and grid for layouts
  • Two-value syntax separates outer and inner display
  • Any element's display type can be changed
