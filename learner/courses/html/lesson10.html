What is Web Accessibility?

Web accessibility (often abbreviated as a11y — "a" followed by 11 letters, then "y") means designing websites that everyone can use, including people with disabilities. This includes people who are blind, have low vision, are deaf, have motor disabilities, or cognitive disabilities. Good accessibility benefits everyone, not just people with disabilities.

Why Accessibility Matters

Important: About 15% of the world's population has some form of disability. Making your site accessible is not just the right thing to do — it's also often a legal requirement and improves SEO.
  • Inclusivity: Everyone deserves equal access to information
  • Legal compliance: Many countries have accessibility laws (ADA, WCAG)
  • Better UX: Accessible sites are easier for everyone to use
  • SEO benefits: Search engines favor accessible content

2. Alt Text for Images

Every image needs alternative text that describes what the image shows. This text is read by screen readers and displayed if the image fails to load.










The diagram shows three servers connected to a load balancer...
Line by line:
  • alt="Bar chart..." — Describes the meaningful information in the image
  • alt="" — Empty alt tells screen readers to skip decorative images
  • aria-describedby — Links to a longer description for complex images

Key ARIA attributes:
  • aria-label — Provides an accessible name when visible text isn't present
  • aria-labelledby — References another element's content as the label
  • aria-hidden — Hides content from assistive technology
  • aria-live — Announces changes to dynamic content
  • aria-expanded — Shows whether a control is expanded or collapsed

5. Keyboard Accessibility

All interactive elements must be accessible via keyboard. Many users navigate using Tab, Enter, and arrow keys.

Keyboard accessibility essentials:
  • tabindex="0" — Makes non-interactive elements focusable
  • role="button" — Tells screen readers this is a button
  • onkeydown — Custom elements need Enter/Space key handling
  • .skip-link — First tab stop should skip repetitive navigation
  • :focus-visible — Shows focus indicator only for keyboard users

8. Testing for Accessibility

Regular testing helps catch accessibility issues before they reach users.
Remember: Automated tools catch only about 30% of accessibility issues. Manual testing and user testing with people with disabilities are essential.

📚 Key Takeaways

  • Use semantic HTML elements instead of generic divs
  • Provide alt text for all meaningful images
  • Label all form inputs properly
  • Ensure keyboard navigation works
  • Meet color contrast requirements (4.5:1 for text)
  • Use ARIA only when HTML isn't sufficient
  • Test regularly with automated and manual methods
