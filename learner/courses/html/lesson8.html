🏷️ Lesson 8: HTML Attributes

Adding extra information to elements

What are Attributes?

Attributes provide additional information about HTML elements. They are always specified in the opening tag and come in name/value pairs.

// Attribute syntax:
<element attribute="value">Contentelement>
         ↑           ↑
       name       value (in quotes)

// Example:
<a href="https://example.com" target="_blank">Linka>

Global Attributes

Global attributes can be used on any HTML element:

id

Unique identifier for an element. Each ID should be unique on the page.

<h1 id="main-title">Welcomeh1>
<div id="header">...div>

// Uses:
// - Link anchors: 
// - CSS styling: #main-title { }
// - JavaScript: document.getElementById("main-title")

class

Used for styling and grouping elements. Multiple classes can be separated by spaces.

// Single class
<p class="highlight">Important textp>

// Multiple classes
<p class="highlight important featured">Very importantp>

// CSS styling:
// .highlight { background: yellow; }
// .important { font-weight: bold; }

id vs class: An id is unique (one per page), like a social security number. A class can be used multiple times, like a category. Use id for unique elements, class for groups.

style

Inline CSS styling (avoid using - prefer external CSS).

<p style="color: purple; font-size: 18px;">Styled textp>

// Use sparingly - external CSS is better for maintainability

title

Tooltip text that appears when hovering over an element.

<p title="This is a tooltip">Hover over me!p>

<abbr title="HyperText Markup Language">HTMLabbr>

data-* Attributes

Store custom data for JavaScript use.

<div data-user-id="12345" 
     data-role="admin">
    User Profile
div>

// Access with JavaScript:
// element.dataset.userId  // "12345"
// element.dataset.role    // "admin"

hidden

Hides the element from view.

<p hidden>This is not visiblep>

// Boolean attribute - no value needed

Boolean Attributes

Some attributes are "boolean" - their presence means true, absence means false.

// These are all the same (true):
<input type="text" disabled>
<input type="text" disabled="disabled">
<input type="text" disabled="">

// To make it false, remove the attribute:
<input type="text">  // enabled

// Common boolean attributes:
disabled   // Disables input
required   // Makes form field required
checked    // Pre-checks checkbox/radio
selected   // Pre-selects option
readonly   // Makes input read-only
autofocus  // Auto-focuses on page load
multiple   // Allows multiple selections

Specific Element Attributes

Link Attributes ()

<a href="https://example.com"    // Destination URL
   target="_blank"            // Open in new tab
   rel="noopener noreferrer" // Security for _blank
   download                   // Download instead of navigate
   hreflang="en"             // Language of linked page
   type="text/html">         // MIME type
   Link text
a>

Image Attributes ()

<img src="photo.jpg"          // Image source (required)
     alt="Description"         // Alt text (required)
     width="300"              // Width in pixels
     height="200"             // Height in pixels
     loading="lazy"           // Lazy load image
     decoding="async">         // Async decoding

Form Input Attributes

<input type="text"           // Input type
       name="username"        // Name for form submission
       value="default"        // Default value
       placeholder="Enter name" // Hint text
       required              // Must be filled
       disabled              // Disabled state
       readonly              // Read-only
       maxlength="20"        // Max characters
       minlength="3"         // Min characters
       pattern="[A-Za-z]+"    // Regex pattern
       autocomplete="off">    // Autocomplete behavior

Form Attributes

<form action="/submit"        // Submit URL
      method="POST"          // HTTP method
      enctype="multipart/form-data" // For file uploads
      novalidate>            // Disable validation
form>

Event Handler Attributes

Attributes that execute JavaScript when events occur (inline event handlers - generally prefer using JavaScript event listeners):

// Common event attributes:

// Mouse events
<button onclick="handleClick()">Clickbutton>
<div onmouseover="highlight()" onmouseout="unhighlight()">...div>

// Form events
<form onsubmit="validate()">...form>
<input onchange="updateValue()">
<input oninput="liveUpdate()">

// Keyboard events
<input onkeydown="handleKey()">

// Load events
<body onload="init()">

Best practice: Instead of inline event handlers, use JavaScript event listeners in your JS files. This keeps your HTML clean and your code maintainable.

Accessibility Attributes

ARIA attributes enhance accessibility for screen readers:

// ARIA role
<div role="navigation">...div>
<button role="switch">...button>

// Labels and descriptions
<button aria-label="Close menu">Xbutton>
<input aria-describedby="password-hint">
<p id="password-hint">Must be 8+ charactersp>

// State attributes
<button aria-expanded="true">Menubutton>
<button aria-pressed="true">Togglebutton>
<div aria-hidden="true">Decorative onlydiv>

// Live regions (announces changes)
<div aria-live="polite">Status messagediv>

Key Takeaways

Remember:

  • Syntax: attribute="value" in opening tag
  • id: Unique identifier (one per page)
  • class: Reusable identifier (many per page)
  • style: Inline CSS (use sparingly)
  • data-*: Custom data for JavaScript
  • Boolean attributes: Present = true, absent = false
  • href/src: URL references
  • alt: Alternative text (accessibility)
  • ARIA: Accessibility enhancement
