📄 Lesson 2: HTML Document Structure

The basic skeleton of every web page

The Basic HTML Template

Every HTML document follows a standard structure. This is the foundation of every webpage you'll create.


<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpagetitle>
head>
<body>
    <h1>Hello, World!h1>
    <p>This is my first webpage.p>
body>
html>

Let's break down each part to understand what it does.

The DOCTYPE Declaration

This tells the browser that this is an HTML5 document. It must be the very first line in your HTML file, before any other code. Without it, browsers may display your page in "quirks mode" (older, inconsistent rendering).

Why it matters: In the past, there were different versions of HTML with different DOCTYPEs. The simple HTML5 DOCTYPE tells modern browsers to use the latest standards.

The html Element

<html lang="en">
    // All page content goes here
html>

The element is the root element that contains all other elements. Everything on your page goes inside it.

The lang Attribute

The lang attribute specifies the language of the content:

  • lang="en" - English
  • lang="es" - Spanish
  • lang="fr" - French
  • lang="zh" - Chinese

Why lang matters: It helps screen readers pronounce content correctly and enables proper spell-checking in browsers.

The head Element

<head>
    // Metadata goes here (not visible on page)
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Titletitle>
    <link rel="stylesheet" href="styles.css">
head>

The contains metadata - information about the page that isn't displayed. This includes:

Meta Tags

// Character encoding (required for special characters):
<meta charset="UTF-8">

// Viewport for responsive design:
<meta name="viewport" content="width=device-width, initial-scale=1.0">

// Description for search engines:
<meta name="description" content="A description of your page">

// Author information:
<meta name="author" content="Your Name">

The title Element

<title>My Awesome Websitetitle>

The title appears in the browser tab and is used by search engines. Make it descriptive!

Linking External Resources

// Link to CSS stylesheet:
<link rel="stylesheet" href="styles.css">

// Link to favicon (tab icon):
<link rel="icon" href="favicon.ico">

The body Element

<body>
    // All visible content goes here
    <h1>Welcome to My Websiteh1>
    <p>This is visible content.p>
    <img src="photo.jpg" alt="A photo">
body>

The contains everything visible on the page: text, images, videos, forms, and more. Everything you want users to see goes here.

head vs body: The head is for metadata (information about the page). The body is for content (what users actually see).

HTML Comments

Comments are notes in your code that aren't displayed. They help document your code.



<p>This text is visible.p>

Use comments to: Explain complex code, leave notes for yourself or others, temporarily disable code during debugging, and document your thought process.

Complete Example

Here's a complete, properly structured HTML document:


<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="A simple webpage">
    <title>My First Webpagetitle>
head>
<body>
    
    <h1>Welcome!h1>
    
    
    <p>This is my first webpage. I'm learning HTML!p>
    
    
    <a href="https://example.com">Click herea>
body>
html>

Key Takeaways

Remember:

  • : Always first, declares HTML5
  • : Root element, use lang attribute
  • : Contains metadata (title, meta tags, links)
  • : Contains all visible content
  • charset="UTF-8": Essential for proper text encoding
  • viewport meta: Critical for mobile responsiveness
  • Comments: Use to document code
