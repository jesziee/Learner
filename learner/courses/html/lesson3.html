📝 Lesson 3: Text Elements

Headings, paragraphs, and text formatting

Headings

HTML has six levels of headings, from

(most important) to

(least important). Headings create a hierarchy for your content.

// All six heading levels:
<h1>Heading 1 - Main Titleh1>
<h2>Heading 2 - Sectionsh2>
<h3>Heading 3 - Subsectionsh3>
<h4>Heading 4 - Minor Sectionsh4>
<h5>Heading 5 - Small Headingsh5>
<h6>Heading 6 - Smallesth6>

Heading Best Practices

  • Use only one

    per page (the main title)

  • Maintain proper hierarchy (don't skip from h1 to h3)
  • Use headings for structure, not just to make text bigger
  • Search engines use headings to understand page content

Semantic meaning: Headings aren't just about size - they tell screen readers and search engines what's important. A proper heading structure improves accessibility and SEO.

Paragraphs

The

element defines a paragraph of text. Browsers automatically add space above and below paragraphs.

<p>This is a paragraph. It contains text that will wrap 
automatically when it reaches the edge of its container.p>

<p>This is another paragraph. Each paragraph gets its own 
spacing from the one above and below it.p>

// Output:
// Two separate blocks of text with space between them

White Space in HTML

HTML collapses multiple spaces and line breaks into a single space:

// This HTML:
<p>This    has    multiple    spacesp>

// Displays as:
This has multiple spaces

// Multiple lines also collapse:
<p>Line one
Line two
Line threep>

// Displays as:
Line one Line two Line three

Why? HTML was designed for structure, not formatting. Use CSS to control spacing and layout, not multiple spaces or empty paragraphs.

to create a line break within a paragraph without starting a new paragraph.

Don't overuse

Text Formatting Elements

HTML provides elements to format text with semantic meaning:

Strong and Emphasis

// Strong importance (typically displays bold)
<strong>Important textstrong>

// Emphasis (typically displays italic)
<em>Emphasized textem>

// Example sentence:
<p>This is <strong>very importantstrong> information.p>
<p>I <em>reallyem> want to go!p>

Presentation Elements (Use Sparingly)

// Bold text (no semantic meaning)
<b>Bold textb>

// Italic text (no semantic meaning)
<i>Italic texti>

// Underlined text
<u>Underlined textu>

// Strikethrough
<s>Strikethrough texts>

Strong vs Bold: means "this is important" (semantic). just means "make this bold" (presentational). Use for accessibility - screen readers may announce it differently.

Other Text Elements

Mark (Highlight)

<p>This is <mark>highlighted textmark>.p>

Small

<p><small>Terms and conditions apply.small>p>

Subscript and Superscript

// Subscript (below baseline)
H<sub>2sub>O    // Water formula

// Superscript (above baseline)
E = mc<sup>2sup>    // Einstein's equation
1<sup>stsup 2<sup>ndsup> 3<sup>rdsup>    // Ordinal numbers

Code Elements

// Inline code
<p>Use the <code>console.log()code> function.p>

// Preformatted text (preserves spaces and line breaks)
<pre>
function greet() {
    console.log("Hello!");
}
pre>

// Keyboard input
<p>Press <kbd>Ctrlkbd> + <kbd>Ckbd> to copy.p>

Blockquote and Quote

// Blockquote (for longer quotes)
<blockquote>
    <p>The only way to do great work is to love what you do.p>
    <footer>— Steve Jobsfooter>
blockquote>

// Inline quote
<p>He said, <q>Hello, world!q> and smiled.p>

// With citation
<blockquote cite="https://example.com/quote-source">
    <p>A famous quote here.p>
blockquote>

Horizontal Rule

The

element creates a thematic break between sections.

<h2>Chapter 1h2>
<p>Content of chapter 1...p>

<hr>

<h2>Chapter 2h2>
<p>Content of chapter 2...p>

Semantic meaning:

isn't just a visual line - it represents a thematic change in content, like a scene change in a story or a transition to a new topic.
