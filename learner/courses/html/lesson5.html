📋 Lesson 5: Lists and Tables

Organizing data with lists and tables

Ordered Lists

Use

    for numbered lists where order matters:

    <ol>
    <li>First itemli>
    <li>Second itemli>
    <li>Third itemli>
ol>

// Output:
1. First item
2. Second item
3. Third item

// Each item is wrapped in 
  1.  (list item)

    2. Ordered List Attributes

    // Change numbering type
<ol type="A">        // A, B, C...
<ol type="a">        // a, b, c...
<ol type="I">        // I, II, III...
<ol type="i">        // i, ii, iii...
<ol type="1">        // 1, 2, 3... (default)

// Start from a different number
<ol start="5">
    <li>This is item 5li>
    <li>This is item 6li>
ol>

// Reverse the order
<ol reversed>
    <li>Thirdli>
    <li>Secondli>
    <li>Firstli>
ol>

Unordered Lists

Use

    for bullet-point lists where order doesn't matter:

    <ul>
    <li>Applesli>
    <li>Orangesli>
    <li>Bananasli>
ul>

// Output:
• Apples
• Oranges
• Bananas

    Changing Bullet Style (with CSS)

    // CSS controls bullet style:
<ul style="list-style-type: disc;">    // • (default)
<ul style="list-style-type: circle;">  // ○
<ul style="list-style-type: square;">  // ■
<ul style="list-style-type: none;">    // No bullets

Nested Lists

Lists can be nested inside other lists:

<ul>
    <li>Fruits
        <ul>
            <li>Applesli>
            <li>Orangesli>
        ul>
    li>
    <li>Vegetables
        <ul>
            <li>Carrotsli>
            <li>Broccolili>
        ul>
    li>
ul>

// Output:
• Fruits
    ○ Apples
    ○ Oranges
• Vegetables
    ○ Carrots
    ○ Broccoli

Important: Nested lists must be inside an

  • element, not directly inside another list element.

    • Description Lists

    Use

    for lists with terms and descriptions:

    <dl>
    <dt>HTMLdt>
    <dd>HyperText Markup Language - the structure of web pagesdd>
    
    <dt>CSSdt>
    <dd>Cascading Style Sheets - styling for web pagesdd>
    
    <dt>JavaScriptdt>
    <dd>A programming language for interactive web pagesdd>
dl>

// dt = description term
// dd = description details

    Basic Table Structure

    Tables organize data in rows and columns:

    <table>
    <tr>                          // Table row
        <th>Nameth>              // Table header cell
        <th>Ageth>
        <th>Cityth>
    tr>
    <tr>
        <td>Johntd>             // Table data cell
        <td>25td>
        <td>New Yorktd>
    tr>
    <tr>
        <td>Janetd>
        <td>30td>
        <td>Los Angelestd>
    tr>
table>

    Table elements:

    - Container for the entire table
    - Table row
    - Table header (bold, centered by default)
    		 - Table data (regular cell)

    Table Sections

    Organize tables into logical sections:

    <table>
    <thead>                      // Table head section
        <tr>
            <th>Nameth>
            <th>Scoreth>
        tr>
    thead>
    <tbody>                      // Table body section
        <tr>
            <td>Alicetd>
            <td>95td>
        tr>
        <tr>
            <td>Bobtd>
            <td>87td>
        tr>
    tbody>
    <tfoot>                      // Table footer section
        <tr>
            <td>Averagetd>
            <td>91td>
        tr>
    tfoot>
table>

    Why use sections? They improve accessibility, help with printing (header repeats on each page), and make styling easier with CSS.

    Spanning Rows and Columns

    Cells can span multiple rows or columns:

    <table>
    <tr>
        <th colspan="3">Quarterly Reportth>  // Spans 3 columns
    tr>
    <tr>
        <th>Q1th>
        <th>Q2th>
        <th>Q3th>
    tr>
    <tr>
        <td rowspan="2">Salestd>    // Spans 2 rows
        <td>$1000td>
        <td>$1200td>
    tr>
    <tr>
        <td>$1100td>
        <td>$1300td>
    tr>
table>

    Table Caption

    Add a title to your table:

    <table>
    <caption>Monthly Sales Reportcaption>
    <thead>
        <tr>
            <th>Monthth>
            <th>Salesth>
        tr>
    thead>
    // ... more rows
table>
