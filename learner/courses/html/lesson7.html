🏗️ Lesson 7: Semantic HTML

Meaningful elements for better structure

What is Semantic HTML?

Semantic HTML uses elements that convey meaning about the content they contain. Instead of using generic containers like

and for everything, semantic elements describe what the content is.

// Non-semantic (doesn't describe content):
<div class="header">...div>
<div class="nav">...div>
<div class="article">...div>
<div class="footer">...div>

// Semantic (describes content):
<header>...header>
<nav>...nav>
<article>...article>
<footer>...footer>

Why semantics matter: They improve accessibility (screen readers understand the page structure), help SEO (search engines better understand content), and make code more readable and maintainable.

Page Structure Elements

Represents introductory content or navigational aids. Can be used for the page header or section headers.

// Page header
<header>
    <h1>My Websiteh1>
    <nav>...nav>
header>

// Article header
<article>
    <header>
        <h2>Article Titleh2>
        <time datetime="2024-01-15">January 15, 2024time>
    header>
    <p>Article content...p>
article>

Contains navigation links. Use for primary navigation, not for all link groups.

<nav>
    <ul>
        <li><a href="/">Homea>li>
        <li><a href="/about">Abouta>li>
        <li><a href="/contact">Contacta>li>
    ul>
nav>

Contains the main content of the page. There should be only one

per page.

<main>
    <h1>Welcome to My Siteh1>
    <p>Main content goes here...p>
main>

// Excludes: header, footer, nav, sidebars

Contains footer content like copyright, contact info, or related links.

<footer>
    <p>© 2024 My Website. All rights reserved.p>
    <nav>
        <a href="/privacy">Privacy Policya>
        <a href="/terms">Terms of Servicea>
    nav>
footer>

Content Sectioning Elements

A thematic grouping of content, typically with a heading.

<section>
    <h2>Our Servicesh2>
    <p>We offer web development, design, and consulting.p>
section>

<section>
    <h2>Contact Ush2>
    <p>Email: hello@example.comp>
section>

Self-contained content that could stand alone (blog post, news article, comment).

<article>
    <header>
        <h2>How to Learn HTMLh2>
        <p>Posted on <time datetime="2024-01-10">January 10time>p>
    header>
    <p>HTML is the foundation of the web...p>
    <footer>
        <p>Written by John Doep>
    footer>
article>

Section vs Article: An article is self-contained and could be syndicated (like a blog post). A section is a thematic grouping within a page. Both typically have headings.

Content tangentially related to the main content (sidebars, pull quotes).

<aside>
    <h3>Related Articlesh3>
    <ul>
        <li><a href="#">CSS Basicsa>li>
        <li><a href="#">JavaScript Introductiona>li>
    ul>
aside>

Other Semantic Elements

<figure>
    <img src="chart.png" alt="Sales chart">

// Machine-readable datetime

// Duration
<time datetime="PT2H30M">2 hours 30 minutestime>

Key Takeaways

Remember:

  • : Introductory content or navigation
  • : Navigation links
  • : Main content (one per page)
  • : Self-contained content
  • : Thematic grouping of content
  • : Related but separate content
  • : Footer content
  • : Machine-readable dates/times
  • Benefits: Better accessibility, SEO, and code readability
