📝 Lesson 6: Forms and Input Elements

Creating interactive forms for user input

The Form Element

The

element is a container for all the inputs that collect user data.

<form action="/submit" method="POST">
    // Form inputs go here
form>

// action: Where to send the data (URL)
// method: How to send it (GET or POST)

// GET: Data visible in URL (search, filters)
// POST: Data hidden in request body (login, forms)

Text Inputs

The most common input type for collecting text:

// Basic text input
<input type="text" name="username">

// With label (important for accessibility)
<label for="username">Username:label>
<input type="text" id="username" name="username">

// With placeholder and required
<input type="text" 
       name="username" 
       placeholder="Enter your username"
       required>

// Password input (characters hidden)
<input type="password" name="password">

// Email input (validates email format)
<input type="email" name="email">

Common Input Attributes

  • name: Identifier for the data (required for form submission)
  • id: Unique identifier (links to label)
  • placeholder: Hint text shown when empty
  • value: Default/pre-filled value
  • required: Makes the field mandatory
  • disabled: Prevents user input
  • readonly: Visible but not editable
  • maxlength: Maximum character count

Labels

Labels are crucial for accessibility and usability:

// Method 1: Using for and id (preferred)
<label for="email">Email Address:label>
<input type="email" id="email" name="email">

// Method 2: Wrapping the input
<label>
    Email Address:
    <input type="email" name="email">
label>

Why labels matter: Clicking a label focuses its input. Screen readers announce the label when the input is focused. Always use labels for accessibility!

Other Input Types

Number Input

<input type="number" 
       name="age" 
       min="0" 
       max="120"
       step="1">

// min/max: Range limits
// step: Increment value (1, 0.5, etc.)

Date and Time Inputs

<input type="date" name="birthday">
<input type="time" name="appointment">
<input type="datetime-local" name="event">
<input type="month" name="expiry">
<input type="week" name="schedule">

Other Useful Types

// Phone number
<input type="tel" name="phone">

// URL
<input type="url" name="website">

// Search field
<input type="search" name="query">

// Hidden field (not visible, but submitted)
<input type="hidden" name="user_id" value="123">

// Color picker
<input type="color" name="favorite" value="#7C3AED">

// Range slider
<input type="range" name="volume" min="0" max="100">

Textarea

For multi-line text input:

<label for="message">Message:label><br>
<textarea id="message" 
          name="message" 
          rows="4" 
          cols="50"
          placeholder="Type your message here...">textarea>

// rows: Height in lines
// cols: Width in characters
// Content goes between opening and closing tags

Checkboxes and Radio Buttons

Checkboxes (Multiple Selections)

<fieldset>
    <legend>Select your interests:legend>
    
    <label>
        <input type="checkbox" name="interests" value="coding"> Coding
    label>
    <label>
        <input type="checkbox" name="interests" value="design"> Design
    label>
    <label>
        <input type="checkbox" name="interests" value="music" checked> Music
    label>
fieldset>

// Same name = same group
// checked = pre-selected

Radio Buttons (Single Selection)

<fieldset>
    <legend>Select your level:legend>
    
    <label>
        <input type="radio" name="level" value="beginner" checked> Beginner
    label>
    <label>
        <input type="radio" name="level" value="intermediate"> Intermediate
    label>
    <label>
        <input type="radio" name="level" value="pro"> Pro
    label>
fieldset>

// Same name = only one can be selected

Buttons

// Submit button (submits the form)
<button type="submit">Submit Formbutton>

// Or using input element
<input type="submit" value="Submit">

// Reset button (clears form)
<button type="reset">Clear Formbutton>

// Regular button (no default behavior)
<button type="button">Click Mebutton>

// Button with icon
<button type="submit">
    <img src="icon.png" alt=""> Submit
button>

Form Validation

HTML5 provides built-in validation:

<form>
    // Required field
    <input type="text" name="name" required>
    
    // Pattern matching (regex)
    <input type="text" 
           name="zip" 
           pattern="[0-9]{5}"
           title="5-digit zip code">
    
    // Min/max length
    <input type="text" 
           name="username" 
           minlength="3" 
           maxlength="20">
    
    // Min/max values
    <input type="number" name="age" min="18" max="100">
    
    <button type="submit">Submitbutton>
form>

Complete Form Example

<form action="/register" method="POST">
    <fieldset>
        <legend>Registration Formlegend>
        
        <label for="name">Full Name:label>
        <input type="text" id="name" name="name" required><br>
        
        <label for="email">Email:label>
        <input type="email" id="email" name="email" required><br>
        
        <label for="password">Password:label>
        <input type="password" id="password" name="password" minlength="8" required><br>
        
        <label for="dob">Date of Birth:label>
        <input type="date" id="dob" name="dob"><br>
        
        <button type="submit">Registerbutton>
    fieldset>
form>
